面向对象编程¶

类与对象的基础语义¶

面向对象编程（OOP）是一种以“对象”为中心的编程范式。在 Python 中，类 是对象的蓝图或模板，对象 是按蓝图创造的实例。如果说列表、字典是 Python 内置的容器，那么类就是你自己发明的新容器——不仅能存数据，还能绑定该数据专属的行为。

定义一个最简单的类¶

class Dog:
    pass

注意

class 关键字定义类，类名通常采用 大驼峰命名法 UpperCamelCase。上面的 Dog 几乎是空的，但已经是合法的类。

创建对象¶

对象是类的实例，通过“调用类名”生成：

my_dog = Dog()
print(type(my_dog))   # <class '__main__.Dog'>

每调用一次，就生成一个独立的新对象（实例）。

类与对象的关系¶

  • 类：描述该类所有实例共享的结构（属性）和行为（方法）。
  • 实例：类的具体化，拥有自己的状态。

Tip

把类想象成造车的图纸，实例则是按图纸下线的真车。图纸只需一份，真车可以有无数辆，它们的颜色、里程可以各自不同，但操作方式（方向盘、油门）都遵照同一张图纸。

方法：定义在类中的函数¶

函数写在类里就叫方法，第一个参数永远是 self（见下一节）。

class Dog:
    def bark(self):
        print("汪汪！")

dog = Dog()
dog.bark()   # 汪汪！

此时 bark 就是 Dog 类的实例方法。虽然直接看起来方法比普通函数多了一个 self，调用时却不用显式传，Python 会自动把调用对象作为第一个参数传入。

__init__ 实例初始化方法¶

对象出生时要初始化自己的数据，这个任务由特殊方法 __init__ 承担。它会在对象创建 之后 立刻调用，是对象设置的入口。

基本使用¶

class Dog:
    def __init__(self, name, age):
        self.name = name          # 把参数绑定到实例属性
        self.age = age

创建对象时传入参数：

d1 = Dog("旺财", 3)
d2 = Dog("来福", 1)

print(d1.name)   # 旺财
print(d2.age)    # 1
  • __init__ 的第一个参数 self 代表当前正在被初始化的实例。
  • 通过 self.属性名 定义的变量是 实例属性，每个实例都有自己的一份，互不干扰。

__init__ 不是构造器¶

严格来说，真正的构造器是 __new__，但 99% 的情况我们只需要 __init__。它负责填充已经创建好的空对象，设置初始状态。不必纠结 __new__，知道有它就好。

设置默认值¶

可以为 __init__ 的参数提供默认值，使某些参数可选。

class Dog:
    def __init__(self, name, age=0):
        self.name = name
        self.age = age

puppy = Dog("小黄")   # age 默认为 0

基本原则

__init__ 里只做简单的属性绑定和校验，避免执行耗时操作（如文件读写、网络请求）。复杂的初始化可以拆出单独的方法，按需调用。

实例属性、类属性与方法的绑定¶

属性和方法都是对象的状态与行为，但它们可以挂在不同的层级上。

实例属性¶

通过 self.xxx 定义在 __init__ 或其他实例方法中的变量，属于每个实例单独所有，互不干扰。

class Cat:
    def __init__(self, name):
        self.name = name

c1 = Cat("咪咪")
c2 = Cat("球球")
c1.name = "大咪"   # 只影响 c1
print(c2.name)      # 球球

实例属性是面向对象封装数据的主要方式。

类属性¶

直接定义在 class 体中的变量，由**所有实例共享**。通过类名或实例均可访问，但如果通过实例对属性赋值，会在实例上创建同名**实例属性**，从而遮蔽类属性。

class Cat:
    species = "猫科"        # 类属性

    def __init__(self, name):
        self.name = name

print(Cat.species)          # 猫科
c = Cat("小黑")
print(c.species)            # 猫科  (通过实例访问)
Cat.species = "猫科动物"
print(c.species)            # 猫科动物（跟着变了）
c.species = "喵星人"        # 实际上给 c 添加了一个实例属性
print(Cat.species)          # 猫科动物（类属性未变）

类属性通常用于定义常量、默认配置、跨实例的计数器等。

方法绑定¶

  • 实例方法：第一个参数为 self，绑定到实例，可以访问实例属性和类属性。
  • 类方法：用 @classmethod 装饰，第一个参数为 cls，绑定到类本身，通常用于工厂方法。
  • 静态方法：用 @staticmethod 装饰，不需要 self 或 cls，就是类的命名空间中的普通函数。
class Demo:
    class_attr = 0

    def instance_method(self):
        return "实例方法", self

    @classmethod
    def class_method(cls):
        return "类方法", cls

    @staticmethod
    def static_method():
        return "静态方法"

三种方法的使用场景泾渭分明：操作实例数据用实例方法；操作类状态或提供替代构造器用类方法；与类和实例无关的工具函数用静态方法。

* 封装约定与@property装饰器¶

封装是 OOP 的核心原则之一：对象应该隐藏内部状态细节，只暴露稳定的接口。Python 没有真正的 private 关键字，但有一套基于约定和名字改编的访问控制。

约定：单下划线 _protected¶

以单个下划线开头的名字表示“受保护的，供内部使用”，外部访问也不会报错，但会明确提醒“不要碰我”。

class BankAccount:
    def __init__(self, owner, balance):
        self._owner = owner         # 受保护属性
        self._balance = balance

这是一种君子协定，所有人都会遵守。

名字改编：双下划线 __private¶

以双下划线开头、最多一个下划线结尾的名字，Python 会自动将其改为 _类名__属性名，从而实现某种程度的私有化，防止意外重名。

class BankAccount:
    def __init__(self, owner, balance):
        self.__owner = owner          # 实际存为 _BankAccount__owner

acc = BankAccount("Ada", 1000)
# print(acc.__owner)    # AttributeError
print(acc._BankAccount__owner)   # 仍可访问，但明显不鼓励

这不是安全机制，只是“改名防手贱”。真要访问谁也拦不住，但至少表示了不该访问的意图。

@property：把方法包装成属性¶

有时我们需要在属性访问时加逻辑（如校验、延迟计算），而又不想改变外部调用方式。@property 装饰器能将一个方法变为“属性”，你依然可以用 obj.attr 的方式读写它，背后其实执行方法。

class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def fahrenheit(self):
        """华氏度，只读属性"""
        return self._celsius * 9 / 5 + 32

t = Temperature(100)
print(t.fahrenheit)   # 212.0，没有括号，像访问属性一样
# t.fahrenheit = 200  # AttributeError: 没有setter

可以进一步给属性加上 setter 和 deleter，实现可控修改。

class Temperature:
    def __init__(self, celsius=0):
        self._celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("温度不能低于绝对零度")
        self._celsius = value

t = Temperature()
t.celsius = 30          # 实际调用 setter
print(t.celsius)        # 30，实际调用 getter

Note

@property 让你以属性访问的简洁外表，保留方法的校验和计算能力，是 Python 封装风格的典范。

* 继承与MRO方法解析顺序¶

继承是在已有类的基础上构建新类的方式。新类（子类）自动获得旧类（父类）的所有属性和方法，并可添加或覆盖。

基本继承¶

class Animal:
    def speak(self):
        print("动物叫")

class Dog(Animal):
    def speak(self):                # 覆盖父类方法
        print("汪汪！")

子类 Dog 继承了 Animal 的所有方法，但 speak 被重写，优先使用自己的版本。

d = Dog()
d.speak()       # 汪汪！

如果子类需要调用父类的同名方法，使用 super()：

class Dog(Animal):
    def speak(self):
        super().speak()    # 先调用父类
        print("汪汪！")

多重继承与MRO¶

Python 支持多继承，一个类可以有多个父类。当调用方法时，解释器按照 方法解析顺序（MRO） 决定从哪个父类继承。MRO 使用 C3线性化 算法，保证一致性，避免菱形继承问题。

class A:
    def method(self):
        print("A")

class B(A):
    def method(self):
        print("B")

class C(A):
    def method(self):
        print("C")

class D(B, C):
    pass

查看 MRO

print(D.__mro__)
# (D, B, C, A, object)
D 的实例调用 method 会找到 B 中的实现。若 B 中未定义，会继续找 C，再找 A，最后从 object 继承。super() 也正是沿着 MRO 链条依次调用。

组合优于继承¶

继承虽有强大复用能力，但过度使用会导致紧耦合。设计时优先考虑 组合，即一个类包含另一个类的实例，通过委托实现功能扩展，往往更灵活。

class Engine:
    def start(self):
        return "引擎启动"

class Car:
    def __init__(self):
        self.engine = Engine()

    def start(self):
        return self.engine.start()  # 委托

Tip

面向对象中有一句名言：“多用组合，少用继承。”它值得深思。

多态与鸭子类型¶

多态（Polymorphism）意味着“同一个接口，不同的行为”。在 Python 中，多态几乎不依赖继承，而是依赖 鸭子类型：一个对象只要实现了某些方法（走路像鸭子、叫起来像鸭子），它就是鸭子。

传统的接口多态¶

class Dog:
    def speak(self):
        return "汪汪"

class Cat:
    def speak(self):
        return "喵"

def animal_sound(animal):
    print(animal.speak())

animal_sound(Dog())   # 汪汪
animal_sound(Cat())   # 喵

无需 Animal 基类，只要对象有 speak() 方法就能工作。这比严格的类型检查更灵活、更符合 Python 哲学。

鸭子类型的实践¶

比如你想写一个函数，接受任何可迭代对象，用 sum() 求其和，完全不需要判断类型。会迭代就是可迭代对象。

def total_sum(data):
    return sum(data)

print(total_sum([1,2,3]))        # 列表
print(total_sum((4,5,6)))        # 元组
print(total_sum({7,8,9}))        # 集合

只要对象满足预期行为（有 __iter__ 方法），代码就正确运行。这就是“如果它走起来像鸭子并且叫起来像鸭子，那它对我们来说就是鸭子”。

何时使用 isinstance() 检查

尽管鸭子类型很强大，有时仍需要在边界处检查类型，尤其是要区分数字和字符串，或者防范非法调用。但不要滥用，避免将程序写成一系列 isinstance 条件链。

Python 的方式是：想检查对象是否实现了某个特定方法，可以尝试调用它然后捕获 AttributeError；或者使用 hasattr。但最直接的方式往往是：信任调用者。

特殊方法：__str__、__repr__、__len__ 等¶

以双下划线开头和结尾的方法被称为 特殊方法（魔法方法），它们定义了对象的“协议”。实现这些方法可以让自定义对象表现得像内置类型。

__str__ 与 __repr__¶

  • __str__：为 print() 和 str() 提供用户友好的字符串表示。
  • __repr__：为解释器和调试提供开发者友好的表示，应该尽量返回能重建对象的代码。如果 __str__ 未定义，会回退到 __repr__。
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

    def __str__(self):
        return f"({self.x}, {self.y})"

p = Point(3, 4)
print(str(p))    # (3, 4)
print(repr(p))   # Point(3, 4)

__len__¶

实现 __len__ 后，对象就能被 len() 测量，并且可以在布尔上下文中判断真假（若 __bool__ 未定义且 __len__ 返回 0 则为假）。

class Stack:
    def __init__(self):
        self._items = []

    def push(self, item):
        self._items.append(item)

    def __len__(self):
        return len(self._items)

s = Stack()
s.push(10)
print(len(s))       # 1
print(bool(s))      # True

其他常用特殊方法¶

  • __getitem__ / __setitem__：支持索引访问，obj[key]。
  • __iter__ / __next__：使对象可迭代，支持 for 循环。
  • __call__：让实例可以像函数一样调用。
  • __eq__：定义 == 相等比较逻辑。
  • __lt__、__gt__：实现 < 和 > 比较，排序时自动使用。
  • __add__、__mul__ 等：重载算术运算符。
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

v1 = Vector(1, 2)
v2 = Vector(3, 4)
v3 = v1 + v2
print(v3.x, v3.y)         # 4 6
print(v1 == Vector(1, 2)) # True

实现了这些特殊方法，你的自定义类就不再孤芳自赏，而是无缝融入 Python 的语言生态，可以与内置类型、标准库函数一起优雅地协作。

本章小结¶

本章你踏入了面向对象的世界：用类和对象组织数据与行为，通过 __init__ 初始化实例，区分实例属性和类属性，用单下划线和 @property 完成优雅封装。继承机制和 MRO 让你在复用代码时有章可循，多态与鸭子类型提醒你保持灵活。最后，特殊方法赋予自定义类与语言原生功能融合的能力。面向对象是构建大型程序的核心思维之一，掌握它之后，你的代码将不仅是逻辑的堆叠，更是可维护、可扩展的有机整体。接下来，我们将深入迭代器和生成器，进一步窥探 Python 懒加载和流式处理的魅力。
