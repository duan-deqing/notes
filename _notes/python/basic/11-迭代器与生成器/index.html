迭代器与生成器¶

可迭代对象与迭代器协议¶

Python 的 for 循环之所以能遍历列表、字符串、字典等如此多样的类型，背后的统一抽象就是 迭代器协议。理解它，你就能写出与 Python 生态无缝衔接的自定义容器，也能更通透地理解生成器的工作原理。

可迭代对象（Iterable）¶

能够返回一个迭代器的对象称为 可迭代对象。它必须实现 __iter__ 方法，该方法被调用时返回一个迭代器。

所有序列类型（列表、字符串、元组、range）、集合、字典、文件对象等都是可迭代对象。通过内置函数 iter() 即可获取一个可迭代对象对应的迭代器：

lst = [1, 2, 3]
it = iter(lst)          # 获取迭代器
print(type(it))         # <class 'list_iterator'>

迭代器（Iterator）¶

迭代器是一个具有状态的流对象，它必须同时实现两个方法：

  • __iter__()：返回自身（因此迭代器本身也是可迭代对象）。
  • __next__()：返回下一个元素；如果没有元素了，则抛出 StopIteration 异常。

正是这个异常，告诉 for 循环该正常结束了。

it = iter([10, 20, 30])
print(next(it))   # 10
print(next(it))   # 20
print(next(it))   # 30
print(next(it))   # StopIteration

for 循环的本质¶

for x in obj:
    process(x)

在内部等价于：

iterator = iter(obj)
while True:
    try:
        x = next(iterator)
    except StopIteration:
        break
    process(x)

这意味着，任何实现了迭代器协议的对象都可以直接放进 for 循环。

自定义可迭代对象与迭代器¶

有时候需要将迭代逻辑独立到一个类中。简单做法是让 __iter__ 返回一个内部迭代器类，或者返回一个生成器（11.2节）。下面是一个计数器的例子：

class Countdown:
    def __init__(self, start):
        self.start = start

    def __iter__(self):
        return CountdownIterator(self.start)

class CountdownIterator:
    def __init__(self, value):
        self.current = value

    def __iter__(self):
        return self

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value

for num in Countdown(5):
    print(num)   # 5, 4, 3, 2, 1

但很多时候，直接把 __iter__ 写成一个生成器函数会更简洁（下一节）。关键是理解迭代器协议：只要你的对象能够按顺序一次次吐出元素，它就能在 for 中运行。

生成器函数与 yield 暂停特性¶

生成器（Generator）是创建迭代器最简洁的方式。它看起来像普通函数，但使用 yield 关键字而不是 return 返回中间值。调用生成器函数时，函数体内的代码 不会立刻执行，而是返回一个生成器对象（一种迭代器）。

yield：暂停并返回值¶

每次调用 next() 或 for 循环迭代时，生成器会从上次暂停的 yield 处继续执行，直到下一个 yield 或函数结束（抛出 StopIteration）。

def simple_gen():
    print("生成器启动")
    yield 1
    print("再次运行")
    yield 2
    print("结束")

gen = simple_gen()
print(next(gen))
# 生成器启动
# 1
print(next(gen))
# 再次运行
# 2
print(next(gen))
# 结束
# StopIteration

生成器是迭代器¶

生成器自动实现了 __iter__ 和 __next__，因此它能直接放入 for 循环：

def count_up_to(max):
    count = 1
    while count <= max:
        yield count
        count += 1

for num in count_up_to(3):
    print(num, end=' ')   # 1 2 3

无限序列与惰性计算¶

生成器最大的威力在于惰性（lazy）：只在需要的时候才计算下一个值，绝不一次性占用巨大内存。这让你可以表达无限序列。

def all_even():
    n = 0
    while True:
        yield n
        n += 2

evens = all_even()
for _ in range(5):
    print(next(evens))   # 0 2 4 6 8

若没有生成器，用列表产生同样序列要么需要无限循环，要么限制长度；生成器则让你以“流”的方式思考，直到你需要才取用。

用生成器实现 __iter__¶

在类中，可以把 __iter__ 定义成一个生成器函数，从而极大地简化自定义可迭代对象。

class Countdown:
    def __init__(self, start):
        self.start = start

    def __iter__(self):
        n = self.start
        while n > 0:
            yield n
            n -= 1

for i in Countdown(3):
    print(i)   # 3, 2, 1

__iter__ 被调用时返回一个生成器对象，完美满足迭代器协议。

yield from：委派子生成器¶

yield from 允许一个生成器把产生值的责任委派给另一个生成器，同时透明地传递 send()、throw() 和 close() 等操作。它常用于将嵌套迭代器展平，或构建生成器链。

def sub_gen():
    yield 'A'
    yield 'B'

def main_gen():
    yield from sub_gen()
    yield 'C'

for val in main_gen():
    print(val)   # A B C

生成器表达式¶

生成器表达式和列表推导式长得几乎一样，唯一区别是把方括号换成圆括号。它们不会立即计算出一个完整列表，而是返回一个生成器，按需产出每一个元素。

squares = (x**2 for x in range(10))
print(squares)      # <generator object <genexpr> at 0x...>
print(next(squares)) # 0
print(next(squares)) # 1

可以像所有生成器一样用在 for 循环或传给 sum、min、max 等需要可迭代对象的函数。

sum(x**2 for x in range(10))   # 285，无需括号也返回生成器（函数调用中可省略外层括号）

与列表推导式的对比¶

  • 列表推导式：立即占用内存生成整个列表。适合需要多次遍历或使用列表方法时。
  • 生成器表达式：惰性产生值，内存占用微小。适合一次性遍历、数据量大、流水线处理。
# 内存对比：假设处理百万数据
import sys
lst = [x for x in range(1_000_000)]
gen = (x for x in range(1_000_000))
print(sys.getsizeof(lst))      # 约 8MB
print(sys.getsizeof(gen))      # 约 200 字节

当只需要迭代一次时，生成器表达式是更优雅和高效的选择。

流水线组合¶

可以将生成器表达式串联起来，构建高效的数据处理管道，每个环节都只在需要时才执行。

lines = (line.strip() for line in open('log.txt', encoding='utf-8'))
errors = (line for line in lines if 'ERROR' in line)
for error in errors:
    print(error)

这里没有一次性读入整个文件，也没有创建临时列表。过滤和清洗在多级生成器中自然流动。

itertools 模块基础应用¶

itertools 是标准库中的迭代器工具集，包含一系列高度优化、专门用于构建高效循环的生成器函数。它被誉为“迭代器的瑞士军刀”。这里介绍最常用的几个。

无穷迭代器¶

  • count(start=0, step=1)：从 start 开始不断递增，可用于枚举或生成无限编号。
  • cycle(iterable)：无限循环一个可迭代对象中的所有元素。
  • repeat(elem, n)：重复元素 n 次；若不指定 n，则无限重复。
from itertools import count, cycle, repeat

# count
for i in count(10):
    if i > 15: break
    print(i)          # 10 11 12 13 14 15

# cycle
for i, color in enumerate(cycle(['red', 'green', 'blue'])):
    if i == 5: break
    print(color)      # red green blue red green

# repeat
print(list(repeat('Hi', 3)))  # ['Hi', 'Hi', 'Hi']

有限迭代器：组合生成¶

  • product(*iterables, repeat=1)：笛卡尔积，计算所有可能的组合（嵌套循环的替代）。
  • permutations(iterable, r)：排列，元素顺序重要。
  • combinations(iterable, r)：组合，无序。
  • combinations_with_replacement(iterable, r)：可重复元素的组合。
import itertools

# 笛卡尔积
suits = ['♠', '♥', '♦', '♣']
ranks = ['A', 'K', 'Q']
for card in itertools.product(ranks, suits):
    print(card)   # ('A', '♠'), ('A', '♥'), ...

# 排列
for p in itertools.permutations('ABC', 2):
    print(p)      # ('A','B'), ('A','C'), ('B','A'), ...

# 组合
for c in itertools.combinations('ABC', 2):
    print(c)      # ('A','B'), ('A','C'), ('B','C')

这些函数返回迭代器，用 list 包裹即可获得列表。

有限迭代器：过滤与分组¶

  • compress(data, selectors)：根据 selectors 中的真值筛选 data。
  • dropwhile(predicate, iterable)：从第一个不满足条件的元素开始返回。
  • takewhile(predicate, iterable)：仅返回满足条件的前缀部分。
  • filterfalse(predicate, iterable)：返回使 predicate 为 False 的元素（filter 的反向）。
  • groupby(iterable, key)：按键函数对连续元素分组。
# dropwhile / takewhile
nums = [0, 2, 4, 6, 5, 7, 9]
print(list(itertools.dropwhile(lambda x: x < 5, nums)))  # [6, 5, 7, 9]
print(list(itertools.takewhile(lambda x: x < 5, nums)))  # [0, 2, 4]

# groupby: 需要先排序才能全局分组
data = [('a',1), ('a',2), ('b',3), ('b',5)]
for key, group in itertools.groupby(data, key=lambda x: x[0]):
    print(key, list(group))
# a [('a',1), ('a',2)]
# b [('b',3), ('b',5)]

连接与展平¶

  • chain(*iterables)：依次连接多个可迭代对象。
  • chain.from_iterable(iterable)：接受一个包含子可迭代对象的迭代器，将其展平。
print(list(itertools.chain([1,2], [3,4])))   # [1,2,3,4]
nested = [[1,2], [3,4], [5]]
print(list(itertools.chain.from_iterable(nested)))  # [1,2,3,4,5]

累积与聚合¶

  • accumulate(iterable, func=operator.add)：滚动聚合，产生每一步的累计值（默认加和）。
  • starmap(func, iterable)：将每一组参数解包后传给 func，相当于 ((func(*args) for args in iterable))。
import operator
nums = [1, 2, 3, 4]
print(list(itertools.accumulate(nums)))          # [1, 3, 6, 10]
print(list(itertools.accumulate(nums, operator.mul)))  # [1, 2, 6, 24]

实用组合：islice、tee、zip_longest¶

  • islice(iterable, start, stop, step)：对迭代器切片，不生成中间列表。
  • tee(iterable, n=2)：复制成 n 个独立的迭代器（内部共享原迭代器）。
  • zip_longest(*iterables, fillvalue=None)：类似 zip，但用填充值补齐最短的可迭代对象。

掌握 itertools 之后，你会发现很多循环可以变成这些标准构件的组合，不但性能更好，也更容易推理和复用。

本章小结¶

迭代器和生成器是 Python 处理数据流的核心抽象。可迭代对象和迭代器协议统一了所有遍历行为；生成器函数用 yield 使状态自动保存，yield from 实现优雅委派；生成器表达式提供内存友好的一站式懒加载；而 itertools 则为复杂迭代逻辑提供了预制的、高效的轮子。现在，你已经拥有处理大规模数据和构建数据处理管道的基础利器。带着这份能力进入应用领域，无论是文件清洗还是实时数据处理，你都能游刃有余。
