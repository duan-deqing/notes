快速开始¶

安装 Python¶

访问Python官网进行下载最新的稳定版，安装过程的几个关键点：

  1. 正确选择安装包，一般都是选择64-bit版本的，具体根据实际情况选择。
  2. 勾选Add Python to PATH，这一步是将Python安装目录和里面的Scrips添加到**系统环境变量**，这确保你能在终端中直接输入python运行。
  3. 安装路径建议是保持默认，但一般可以放在自己的开发环境目录，统一管理。

验证安装¶

打开终端(Windows: Powershell / CMD; MacOS/Linux： Terminal)，输入：

`
python --version

`
python3 --version

显示出类似 Python 3.12.3 的信息，则表示安装成功。若提示“不是内部或外部命令”，请检查 PATH 配置。

版本管理工具¶

为什么要版本管理工具？

当你的电脑上同时存在多个 Python 项目，且它们需要不同的 Python 版本时，手动切换会非常痛苦。

主流的Python版本管理方案有下面两种：

  • pyenv : 轻量级，可以为每个目录设定独立的 Python 版本，使用命令行切换比较方便。
  • Anaconda / Miniconda : 集成了 Python、conda 包管理器和大量常用科学包，虚拟环境管理功能强大。

运行第一个程序¶

编写一个 python 程序，再终端输出 Hello World!

编写代码¶

用任意文本编辑器（如 VS Code、Sublime Text，甚至记事本）新建一个文件，命名为 hello.py，输入下面这一行代码：

print("Hello World!")

运行程序¶

打开终端，切换到文件所在目录，执行：

python hello.py
终端会输出

Hello, World!

注释、缩进与PEP8¶

注释¶

  • 单行注释 : 用井号 # 开头，从 # 开始到行尾均被视为注释。
# 这是一条注释
x = 10 # 这也能写注释
  • 多行注释 : Python 没有专门的多行注释符号，通常使用由三个引号（单引号或双引号）括起来的多行字符串来充当。
"""
这是多行注释的常规写法。
可以跨越多行。
"""

注意

严格来说它是个未赋值给变量的字符串常量，解释器会生成它然后丢弃。

注释的原则

注释应该解释 “为什么”（Why）这样做，而不是 “是什么”（What）。代码本身应该清晰到足以说明做了什么。对于复杂的算法、业务逻辑、临时解决措施，写上注释是一种美德；对一目了然的操作画蛇添足则是噪声。

缩进¶

大多数编程语言使用花括号 {} 或 begin/end 关键字来划分代码块，而 Python 用缩进。这是一个强制性的语法要求：同一代码块内的语句必须有相同的缩进长度，通常约定使用4个空格。

if x > 0:
    print("正数")      # 缩进4个空格，表示 if 块的一部分
    y = x
else:
    print("非正数")

注意

缩进必须是“一致”的，不能混合使用 Tab 和空格。

PEP8初识¶

PEP 是“Python Enhancement Proposal”（Python 增强提案）的缩写，其中 PEP 8 是 Python 社区公认的编码风格指南。它不是强制规范，但遵循它能让你的代码与绝大多数 Python 程序员保持一致的风格，提高可读性和可维护性。以下是几个初学就要注意的点：

命名¶

  • 变量名、函数名：小写字母，单词间用下划线连接（user_name、calculate_area）
  • 常量：全大写字母，用下划线连接（MAX_SIZE、DEFAULT_HOST）
  • 类名：骆驼命名法，即每个单词首字母大写（DataLoader、HttpClient）

空格¶

  • 逗号、冒号后加一个空格，前面不加。
  • 运算符两边各加一个空格：a = b + c 而不是 a=b+c。
  • 函数名与左括号之间不加空格：print(x)，不是 print (x)。
print(1, 2, 4)
a = 1 + 3
print(x)

每行长度¶

建议最长不超过 79 个字符，文档字符串或注释不超过 72 个字符。可以续行时用小括号、中括号或反斜杠。

导入¶

每个导入独占一行，按“标准库 -> 第三方库 -> 本地库”分组，每组之间空一行。

# 标准库
import os

# 第三方库
from <第三方库> import xxxx

# 本地库
from <本地库> import xxx

空白行¶

顶层函数和类定义之间用两个空行分隔；类内方法定义之间用一个空行分隔。
