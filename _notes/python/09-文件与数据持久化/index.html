文件与数据持久化¶

文件打开模式与编码声明¶

文件是操作系统管理持久数据的基本单元。Python 通过内置的 open() 函数建立程序与外部文件的桥梁，返回一个文件对象，然后你可以读、写或添加内容。

open() 的基本签名¶

open(file, mode='r', buffering=-1, encoding=None, errors=None, newline=None, closefd=True, opener=None)

最常用的参数是 file（文件路径）和 mode（打开模式），以及 encoding。

核心打开模式¶

模式字符 含义
'r' 只读。文件必须存在，指针在开头（默认模式）
'w' 只写。若文件存在则清空，不存在则创建
'a' 追加写。文件存在则将指针移到末尾，不存在则创建
'x' 独占创建。若文件已存在则抛出 FileExistsError
'b' 二进制模式。用于非文本文件（如图片、音频），组合使用如 'rb', 'wb'
't' 文本模式（默认）。自动处理换行符转换和编码解码
'+' 读写模式。如 'r+' 可同时读写，文件必须存在；'w+' 写入并读取，会清空文件

常见的组合：

f = open('data.txt', 'r')           # 文本只读
f = open('report.txt', 'w')         # 文本只写
f = open('log.txt', 'a')            # 文本追加
f = open('image.png', 'rb')         # 二进制读

编码声明的重要性

文本模式下，encoding 参数指定字符编码。若不指定，Python 会使用系统默认编码（Windows 下可能是 cp936/gbk，Linux/macOS 下通常是 utf-8）。这种跨平台的不一致是乱码的头号来源。永远显式指定 encoding='utf-8'，可以避免 99% 的编码问题。 

# 推荐写法
f = open('data.txt', 'r', encoding='utf-8')
处理非 UTF-8 文件时，指定对应编码；如果遇到无法解码的字节，可通过 errors 参数控制行为（如 'ignore' 跳过，'replace' 替换为�）。

文件对象是可迭代的¶

对于文本文件，文件对象本身就是一个可迭代的“行生成器”，可以用 for 循环逐行读取，这是最内存友好的方式。

f = open('poem.txt', 'r', encoding='utf-8')
for line in f:
    print(line, end='')
f.close()

手动关闭

调用 f.close() 释放操作系统资源。但若在读写过程中发生异常，close() 可能被跳过。下一节会看到 with 语句如何完美解决这个问题。

文本文件的读、写与指针操作¶

读取方法¶

  • read(size=-1)：读取指定数量的字符（文本模式）或字节（二进制模式）。省略或负数则读取全部内容，需小心大文件会撑爆内存。
  • readline()：读取一行，包括行尾的换行符；如果到了文件末尾，返回空字符串。
  • readlines()：一次性读取所有行，返回列表，每行带换行符。同样不适用于大文件。
  • 迭代器方式：直接遍历文件对象，一次只加载一行到内存，高效安全。
with open('story.txt', 'r', encoding='utf-8') as f:
    for line in f:
        # 处理每一行，line 带有换行符
        print(f"行内容: {line.rstrip()}")

写入方法¶

  • write(s)：将字符串 s 写入文件。需自己添加换行符 '\n'。
  • writelines(seq)：写入一个序列的字符串，不会自动添加换行符。
lines = ['第一行\n', '第二行\n', '第三行\n']
with open('output.txt', 'w', encoding='utf-8') as f:
    f.writelines(lines)

文件指针¶

每个打开的文件都有一个“当前位置”指针，指示下一次读或写从哪里开始。

  • tell()：返回当前指针的字节偏移量（文本模式下可能因换行符转换而不准确，建议在二进制模式使用）。
  • seek(offset, whence=0)：移动指针到指定位置。
  • whence=0：从文件开头偏移（offest 通常为正）。
  • whence=1：从当前位置偏移。
  • whence=2：从文件末尾偏移，offset 通常为负。

在文本模式下，seek 只允许回到文件开头，或者用 f.seek(0, 2) 移到末尾绑定偏移量。绝对定位在文本模式因编码问题不可靠，因此涉及指针操作常以二进制模式打开。

追加模式的指针¶

以 'a' 打开时，所有写入都发生在文件末尾，即使你 seek 到前面，写入时指针会自动移回末尾。这保证了追加写不会覆盖旧数据。

with 语句与上下文管理器协议¶

with 语句专为资源管理设计，能确保无论是否发生异常，资源都会被正确释放。在文件操作中，它 可以自动调用 close()。

基本用法¶

with open('data.txt', 'r', encoding='utf-8') as f:
    content = f.read()
# 退出 with 块后，f 自动关闭

as 后获取的文件对象只在 with 块内有效。块结束时，解释器调用文件对象的 __exit__() 方法完成清理，即使块内发生了异常也会执行。

同时管理多个文件¶

可以用逗号分隔：

with open('src.txt', 'r', encoding='utf-8') as f_in, \
     open('dst.txt', 'w', encoding='utf-8') as f_out:
    for line in f_in:
        f_out.write(line.upper())

Python 3.10+ 还支持用括号多行写法

with (
    open('a.txt', 'r') as fa,
    open('b.txt', 'r') as fb
):
    ...

上下文管理器协议¶

任何实现了 __enter__() 和 __exit__(exc_type, exc_val, exc_tb) 方法的对象都可以使用 with。open() 返回的文件对象就是一个上下文管理器。

你可以自定义上下文管理器来管理锁、数据库连接等资源。例如，一个简单的计时器：

import time

class Timer:
    def __enter__(self):
        self.start = time.perf_counter()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        elapsed = time.perf_counter() - self.start
        print(f"耗时: {elapsed:.4f} 秒")
        # 返回 False 会传播异常，True 则吞掉异常（一般不这么做）

with Timer():
    # 耗时操作
    sum(range(10_000_000))

__exit__ 方法会在块结束时执行，三个参数分别代表异常类型、值和回溯对象；如果没有异常，它们都是 None。

对比传统的 try / finally¶

# 旧方式
f = open('data.txt', 'r', encoding='utf-8')
try:
    data = f.read()
finally:
    f.close()

# 新方式
with open('data.txt', 'r', encoding='utf-8') as f:
    data = f.read()

Note

with 少写两行，且不会遗忘 close()。处理文件的铁律：永远使用 with，不手动调用 close()。

CSV 格式解析与 csv 模块¶

CSV（逗号分隔值）是表格数据最通用的交换格式，几乎被所有电子表格和数据库支持。Python 的 csv 模块能正确处理引号、转义、换行等复杂情况，远比手动 split(',') 安全。

读取 CSV 文件¶

csv.reader：返回一个可迭代的读取器，每行是一个字符串列表。

import csv

with open('students.csv', 'r', encoding='utf-8') as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)   # ['Alice', '95', 'A']

如果 CSV 首行是标题，可以单独取出或跳过：

with open('students.csv', 'r', encoding='utf-8') as f:
    reader = csv.reader(f)
    header = next(reader)
    for row in reader:
        print(dict(zip(header, row)))

csv.DictReader：直接将每行解析为字典，键为标题行。

with open('students.csv', 'r', encoding='utf-8') as f:
    reader = csv.DictReader(f)
    for record in reader:
        print(record['name'], record['score'])

写入 CSV 文件¶

csv.writer：

data = [
    ['name', 'age', 'city'],
    ['Ada', '25', 'Hangzhou'],
    ['Bob', '30', 'Beijing']
]

with open('output.csv', 'w', newline='', encoding='utf-8') as f:
    writer = csv.writer(f)
    writer.writerows(data)

极其重要的一点：打开文件时传入 newline=''，让 csv 模块完全控制换行，避免在 Windows 上出现多余空行。

csv.DictWriter：写入字典列表，必须提供字段名。

records = [
    {'name': 'Ada', 'score': 95},
    {'name': 'Bob', 'score': 87}
]

with open('output.csv', 'w', newline='', encoding='utf-8') as f:
    writer = csv.DictWriter(f, fieldnames=['name', 'score'])
    writer.writeheader()
    writer.writerows(records)

自定义方言与参数¶

CSV 的“逗号”可能被换成分号、制表符等。通过 delimiter 参数指定分隔符，通过 quotechar 指定引号字符，quoting 控制引用行为。

# 读取制表符分隔的文件
reader = csv.reader(f, delimiter='\t')

# 写入时所有非数字字段都用引号括起来
writer = csv.writer(f, quoting=csv.QUOTE_NONNUMERIC)

掌握 csv 模块，日常表格数据清洗将不在话下。

JSON序列化与反序列化¶

JSON（JavaScript Object Notation）是跨语言数据交换的事实标准，几乎成了 Web API 的通用语。Python 的 json 模块可以在 Python 对象和 JSON 字符串之间轻松转换。

核心对应关系¶

Python 类型 JSON 类型
dict object
list, tuple array
str string
int, float number
True, False true, false
None null

序列化（Python → JSON）¶

  • json.dumps(obj)：将 Python 对象转换为 JSON 字符串。
  • json.dump(obj, fp)：直接写入文件。
import json

data = {
    "name": "Ada",
    "age": 25,
    "is_student": False,
    "courses": ["Math", "CS"],
    "address": None
}

# 转换为字符串
json_str = json.dumps(data, ensure_ascii=False, indent=2)
print(json_str)

# 写入文件
with open('data.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

参数说明：

  • ensure_ascii=False：允许直接输出非 ASCII 字符（如中文），而不是 \uXXXX 转义。
  • indent=2：美化输出，指定缩进空格数。
  • sort_keys=True：按键排序输出，便于版本比较。

反序列化（JSON → Python）¶

  • json.loads(s)：从字符串解析。
  • json.load(fp)：从文件对象解析。
# 从字符串
json_str = '{"name": "Ada", "age": 25}'
data = json.loads(json_str)
print(data['name'])   # Ada

# 从文件
with open('data.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

处理自定义对象¶

直接对自定义类的实例进行序列化会抛出 TypeError。解决方案有两种：

  1. 传入 default 参数，将不可序列化的对象转换为内置类型。

    from datetime import datetime

def default_serializer(obj):
    if isinstance(obj, datetime):
        return obj.isoformat()
    raise TypeError(f"无法序列化类型 {type(obj)}")

data = {'time': datetime.now()}
json_str = json.dumps(data, default=default_serializer, ensure_ascii=False)

  2. 自定义编码器：继承 json.JSONEncoder 并重写 default 方法，然后传入 cls 参数。

反序列化回自定义对象¶

loads 的 object_hook 参数可以接受将字典转换为自定义对象。

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

def as_person(d):
    return Person(d['name'], d['age'])

json_str = '{"name": "Ada", "age": 25}'
p = json.loads(json_str, object_hook=as_person)
print(p.name, p.age)   # Ada 25

注意事项

  • JSON 不支持注释、尾随逗号，这是严格格式。
  • 日期时间等类型没有原生表示，需要约定序列化格式（如 ISO 8601）。
  • 不要试图序列化包含循环引用的对象，会导致递归错误。
  • 当数据量大时，优先用文件接口 json.dump() / json.load() 而非字符串版本，节省内存。

掌握了 JSON 和 CSV，就打通了程序与外界交换数据的两条主干道。

本章小结¶

文件操作让程序拥有了跨越运行的“记忆”。你学会了使用 open() 的各种模式，用 with 优雅地管理资源，利用文件指针进行精细控制，还掌握了 CSV 和 JSON 这两种最通用的数据持久化格式。从这章开始，你的代码已经可以处理真实世界的数据了。下一章，我们将进入面向对象编程，用类和对象构建更高层次的组织结构。
