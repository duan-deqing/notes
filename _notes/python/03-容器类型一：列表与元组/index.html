容器类型一：列表与元组¶

列表的创建与特性¶

列表（list）是 Python 中最灵活、最常用的内置容器之一。你可以把它想象成 一列有编号的储物柜：每个柜子里可以放任何东西，柜子大小自动适配，而且可以随时增删调换。

创建列表¶

使用方括号 [] 创建列表，元素之间用逗号分隔。

empty = []                     # 空列表
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.14, True]  # 不同类型可以混放

列表中的元素可以是任何 Python 对象，包括另一个列表（嵌套列表）：

matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

也可以用内置函数 list() 将其他可迭代对象（如字符串、元组、range 对象）转换为列表。

list("abc")        # ['a', 'b', 'c']
list((1, 2, 3))    # [1, 2, 3]
list(range(5))     # [0, 1, 2, 3, 4]

列表的核心特性¶

重点理解

  • 有序：元素按照插入顺序存放，每个元素都有固定的位置（索引）。
  • 可变：列表的内容可以原地修改——增、删、改都直接作用在原列表上。
  • 可包含重复值：同一个值可以出现多次。
  • 动态：大小不是固定的，添加元素会自动扩容。
nums = [10, 20, 30]
nums[0] = 99          # 修改：列表变为 [99, 20, 30]
nums.append(40)       # 追加：变为 [99, 20, 30, 40]

注意

列表的“可变”特性要注意：两个变量指向同一个列表对象时，修改一个会影响另一个。 

a = [1, 2, 3]
b = a                 # b 指向同一个列表
b.append(4)
print(a)              # [1, 2, 3, 4]
如果想让 b 成为一个独立的拷贝，需要用 b = a.copy() 或 b = a[:]，稍后会详细展开

列表的索引与切片进阶¶

列表的索引和切片与字符串完全一致，但因为列表可变，切片不仅可以读取子序列，还能用来 原地修改部分元素。

索引¶

从 0 开始递增，[-1] 访问最后一个元素。越界会抛出 IndexError。

fruits = ["苹果", "香蕉", "橙子", "葡萄"]
fruits[0]     # "苹果"
fruits[-1]    # "葡萄"

切片基本用法¶

list[start:end:step] 返回一个 新列表，原列表不受影响。所有参数都可省略。

nums = [0, 1, 2, 3, 4, 5]
nums[2:5]     # [2, 3, 4]  索引 2,3,4
nums[:3]      # [0, 1, 2]
nums[3:]      # [3, 4, 5]
nums[::2]     # [0, 2, 4]   隔一个取一个
nums[::-1]    # [5, 4, 3, 2, 1, 0]  反转

切片进阶：赋值与删除¶

切片不仅能读，还能作为 左值 进行赋值。这使得你可以替换、插入或删除多个元素。

替换一段：

nums = [0, 1, 2, 3, 4, 5]
nums[1:3] = [10, 20]   # 索引1和2被替换为 10 和 20
# nums 变成 [0, 10, 20, 3, 4, 5]

替换的切片长度和新赋值的列表长度 不必相等，列表会自动伸缩：

nums[1:3] = [100]      # 用1个元素替换原来的2个元素
# 结果：[0, 100, 3, 4, 5]

nums[2:2] = [8, 8]     # 在索引2处插入（切片为空，新元素插入该位置）
# 结果：[0, 100, 8, 8, 3, 4, 5]

删除一段：

nums[1:4] = []         # 将索引1到3的元素替换为空，即删除
# 或者使用 del nums[1:4]

这种切片赋值的灵活性是 Python 列表的一大特色，可以用很简洁的语法实现复杂修改。

浅拷贝提醒

切片读写都是浅拷贝：如果列表中包含可变对象（如子列表），复制出的新列表仍指向相同的子对象。这个概念在后续深入时会详细展开。

列表常用方法：增删改查与排序¶

列表对象自带一系列高效方法，记住它们能让你操作列表时如臂使指。

增加元素¶

  • append(x)：在 末尾 添加一个元素，最常用。
  • insert(i, x)：在指定索引 i 处插入元素，原元素及之后的后移。
  • extend(iterable)：将可迭代对象中的每个元素逐个追加到末尾，用于合并列表。
items = [1, 2]
items.append(3)       # [1, 2, 3]
items.insert(1, 99)   # [1, 99, 2, 3]
items.extend([4, 5])  # [1, 99, 2, 3, 4, 5]

注意

append 和 extend 的区别易混淆：append 把参数当成一个整体元素添加；extend 把参数拆开逐个加入。 

a = [1]
a.append([2, 3])   # [1, [2, 3]]
a.extend([2, 3])   # [1, 2, 3]

删除元素¶

  • pop(i=-1)：移除并 返回 索引 i 处的元素，默认移除最后一个。
  • remove(x)：移除第一个值等于 x 的元素，找不到则 ValueError。
  • clear()：清空整个列表。
fruits = ["苹果", "香蕉", "橙子", "香蕉"]
last = fruits.pop()      # "香蕉"，fruits 变为 ["苹果", "香蕉", "橙子"]
first = fruits.pop(0)    # "苹果"
fruits.remove("香蕉")    # 移除第一个 "香蕉"
fruits.clear()           # []

也可以用 del 语句按索引或切片删除：

del items[2]        # 删除索引2的元素
del items[1:3]      # 删除一段

修改与查找¶

  • 直接通过索引赋值修改。
  • index(x, start, end)：返回第一个值为 x 的元素索引，可指定查找范围，找不到则 ValueError。
  • count(x)：返回值为 x 的元素在列表中出现的次数。
nums = [10, 20, 30, 20]
nums.index(20)       # 1
nums.index(20, 2)    # 3 （从索引2开始搜索）
nums.count(20)       # 2

排序与反转¶

  • sort(key=None, reverse=False)：原地排序，修改原列表。默认升序，reverse=True 降序。key 可指定一个函数，用于提取比较键。
  • sorted(iterable, key=None, reverse=False)：内置函数，返回新排序后的列表，原列表不变。
  • reverse()：原地反转列表。
nums = [3, 1, 4, 1, 5]
nums.sort()                  # [1, 1, 3, 4, 5]
nums.sort(reverse=True)      # [5, 4, 3, 1, 1]

words = ["apple", "banana", "cherry"]
words.sort(key=len)          # 按长度排序：["apple", "cherry", "banana"]

# sorted 保持原样
original = [3, 1, 2]
new_list = sorted(original)  # [1, 2, 3]，original 仍是 [3, 1, 2]

注意

sorted() 适用于任何可迭代对象，sort() 只有列表才有。

---¶

元组：不可变序列的定义与用途¶

元组（tuple）是列表的表亲，最大的区别就是：元组一旦创建，里面的元素不能增、不能删、不能改。它是一种不可变序列。

创建元组¶

使用圆括号 ()，元素之间用逗号隔开。

t1 = ()                 # 空元组
t2 = (1, 2, 3)
t3 = 1, 2, 3           # 甚至可以省略括号，逗号才是关键
t4 = ("hello", 3.14)
t5 = (42,)              # 单元素元组：逗号绝对不能省！

特别注意

(42) 只是一个加了括号的整数，不是元组。(42,) 才是单元素元组。 也可以用 tuple() 将其他可迭代对象转换为元组： 

tuple([1, 2, 3])        # (1, 2, 3)
tuple("abc")            # ('a', 'b', 'c')

为什么需要不可变性？

  • 安全性：你不想某些数据被意外修改时，元组是天然的保护。
  • 可哈希：不可变性意味着元组可以作为字典的键或集合的元素，而列表不行。
  • 性能：元组的结构比列表更简单，创建和遍历速度稍微快一点。
  • 表达语义：使用元组通常隐含表示这组数据是一个整体、有固定结构（如坐标 (x, y)、颜色 (r, g, b)）。

* 元组的“不可变”是浅层的¶

如果元组中包含可变对象（如列表），元组自身结构不变（你不能增删元素），但那个可变对象的内容仍然可以修改。

t = (1, [2, 3], 4)
t[1].append(99)   # t 变成 (1, [2, 3, 99], 4)，因为列表内容被改了
t[0] = 10         # TypeError! 不可修改元素引用

所以严格地说，元组不可变的是 元素对象的引用，而不是对象本身。

===¶

序列通用操作与成员检测¶

列表和元组都属于 Python 的 序列类型，共用一套通用操作。这些操作也适用于字符串。

通用序列操作¶

  • len(seq)：返回序列长度（元素个数）。
  • min(seq) / max(seq)：返回最小/最大元素，要求元素间可比较且序列非空。
  • sum(seq, start=0)：求和，元素必须为数字。
  • x in seq / x not in seq：成员检测，检查 x 是否在序列中出现。
  • seq1 + seq2：连接两个 同类型 序列（列表+列表，元组+元组，字符串+字符串）。
  • seq * n 或 n * seq：重复序列内容 n 次。
nums = [3, 1, 4, 1, 5]
len(nums)      # 5
min(nums)      # 1
max(nums)      # 5
sum(nums)      # 14
3 in nums      # True
[1,2] + [3,4]  # [1, 2, 3, 4]
[0] * 5        # [0, 0, 0, 0, 0]

注意

+ 和 * 操作会产生全新的对象，不会修改原序列。对于包含可变元素的嵌套序列，* 操作是浅拷贝。 

row = [0] * 3            # [0, 0, 0]
matrix = [row] * 3       # 三行，但都指向同一个 row 列表
matrix[0][0] = 99
print(matrix)            # [[99, 0, 0], [99, 0, 0], [99, 0, 0]] —— 全部变了！
创建独立行的正确方式是列表推导式，稍后介绍。

成员检测效率¶

列表和元组的 in 操作需要 遍历整个序列 比较，时间复杂度 O(n)。如果成员检测是你的高频操作，应该考虑使用集合（set）或字典键，它们的平均复杂度是 O(1)。

序列比较¶

同类型的序列支持 逐项比较（字典序）。

(1, 2, 3) < (1, 2, 4)      # True
[1, 2] < [1, 2, 0]         # True (前面都相等，长的更大？实际上短的小)
# 比较规则：先比较对应元素，如果都相等则长度短的较小

解包赋值与多变量交换¶

解包（unpacking）是 Python 处理序列时极具表现力的特性，让你能一次性把序列中的元素赋给多个变量。

基本解包¶

等号左侧是一组变量，右侧是一个长度匹配的序列。

point = (3, 5)
x, y = point          # x=3, y=5

data = ["Alice", 25, "Engineer"]
name, age, job = data  # name="Alice", age=25, job="Engineer"

如果左右数量不匹配，会引发 ValueError。

* 扩展解包（Python 3 新增）¶

使用 * 前缀的变量可以吸收剩余元素，形成列表。

first, *rest = [1, 2, 3, 4, 5]
# first=1, rest=[2,3,4,5]

*begin, last = (1, 2, 3, 4)
# begin=[1,2,3], last=4

a, *middle, b = "hello"
# a='h', middle=['e','l','l'], b='o'

这在分离头尾、丢弃不定量元素时非常有用。

filename, *extensions = "report.final.pdf".split(".")
# filename="report", extensions=["final", "pdf"]

多变量交换¶

正是解包语法让交换两个变量的值变成一行，且不需要临时变量：

a, b = b, a

原理是右侧先构造一个元组 (b, a)，然后解包赋值给左侧 a, b。整个过程原子且清晰。

利用解包遍历¶

在 for 循环中结合 enumerate() 或 zip() 时，可以解包出多个值：

pairs = [(1, 'a'), (2, 'b'), (3, 'c')]
for num, letter in pairs:
    print(num, letter)

忽略部分值¶

Note

如果某些值不关心，可以用 _ 作为变量名（这是约定俗成的占位符）。 

_, _, important = (10, 20, 30)   # important=30

本章小结¶

列表和元组是 Python 中最核心的序列容器。列表提供了极大的灵活性——能改能增能删；元组则用不可变性换来了安全与可哈希。掌握它们的索引切片、通用操作和解包技巧，你就拥有了一把处理成批数据的瑞士军刀。接下来，我们将进入另一种维度的容器：字典与集合，它们不再靠位置，而是靠关联来组织信息。
