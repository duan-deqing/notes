# 流程控制：条件与循环

## 布尔类型与比较运算符

> 程序要想做出决策，离不开“真”与“假”。Python 用 `bool` 类型表达这种两值逻辑，它只有两个实例：`True` 和 `False`（首字母必须大写）。这看似简单，却是所有分支和循环的基石。

### 布尔值的字面量

```python
flag = True
done = False
print(type(flag))   # <class 'bool'>
```

在 Python 内部，`bool` 实际上是 `int` 的子类——`True` 等价于 `1`，`False` 等价于 `0`，甚至可以进行算术运算（但别真这么干，会让人困惑）。

```python
True == 1    # True
False == 0   # True
True + True  # 2 （诡异但合法）
```

### 比较运算符

比较运算的结果就是布尔值。Python 提供了一系列比较运算符：

| 运算符   | 含义                 | 示例               |
| :------- | :------------------- | :----------------- |
| `==`     | 等于（值相等）       | `5 == 5` → `True`  |
| `!=`     | 不等于               | `5 != 3` → `True`  |
| `<`      | 小于                 | `3 < 5` → `True`   |
| `>`      | 大于                 | `5 > 3` → `True`   |
| `<=`     | 小于等于             | `3 <= 5` → `True`  |
| `>=`     | 大于等于             | `5 >= 5` → `True`  |
| `is`     | 同一对象（身份相等） | `a is b`           |
| `is not` | 不是同一对象         | `a is not b`       |
| `in`     | 成员检测             | `"a" in "abc"`     |
| `not in` | 非成员               | `"z" not in "abc"` |

### 比较中的两个“等于”

`==` 检查的是值是否相同，而 `is` 检查的是两个变量是否指向内存中同一个对象。绝大多数情况下，我们关心的是值，所以 `==` 使用频率远高于 `is`。`is` 主要用于和 `None` 的比较（公认写法 `x is None`），因为 `None` 是单例。

```python
a = [1, 2, 3]
b = [1, 2, 3]
a == b   # True  （值相同）
a is b   # False （两个独立的列表对象）
```

### 比较链

Python 允许将多个比较串联在一起，数学味十足：

```python
x = 10
1 < x <= 20    # True，等价于 (1 < x) and (x <= 20)
```

这种写法不但可读性高，还能保证中间表达式只被求值一次。

## 逻辑运算符：and、or、not

> 逻辑运算符用于组合或翻转布尔值，是构造复杂条件的基本工具。Python 使用英文单词而非符号（`&&`、`||`、`!`），读起来更像自然语言。

### 基本语义

| 运算符 | 含义                              | 示例                 |
| :----- | :-------------------------------- | :------------------- |
| `and`  | 与：两边都为 `True` 才为 `True`   | `x > 0 and x < 10`   |
| `or`   | 或：至少一边为 `True` 即为 `True` | `x < 0 or x > 10`    |
| `not`  | 非：取反                          | `not True` → `False` |

### 短路逻辑

这是提升效率和编写安全代码的关键特性：Python 在计算 `and` 或 `or` 时，如果已经能从左边推出最终结果，就 **不再计算右边**。

- `a and b`：如果 `a` 为假，直接返回 `a`（假值），不碰 `b`。
- `a or b`：如果 `a` 为真，直接返回 `a`（真值），不碰 `b`。

正因为短路特性，我们可以写出安全的“先检查后使用”模式：

```python
# 若 lst 为空，短路避免索引越界
if len(lst) > 0 and lst[0] == 10:
    print("首元素是10")
```

上面代码里，当 `len(lst) > 0` 为 `False` 时，`and` 直接短路，右边的 `lst[0]` 不会执行，从而避免了 `IndexError`。

### 真假值与“真值测试”

除了 `True` 和 `False`，所有 Python 对象都可以用在需要布尔值的上下文（如 `if` 语句）中。以下值被视为 **“假”**：

- `False`、`None`
- 数值零：`0`、`0.0`、`0j`
- 空序列或容器：`""`、`[]`、`()`、`{}`、`set()`、`range(0)`
- 自定义类型中实现 `__bool__` 或 `__len__` 返回假值的对象

其余一切对象都被视为 **“真”**。这种机制称为 **真值测试（Truthy/Falsy）**。它让代码极其简洁，但需注意区分“空”和“不存在”。

```python
name = ""
if not name:            # 等价于 if name == ""
    print("名字为空")
```

## 条件语句：if-elif-else结构

> 条件语句让程序能根据不同的情况执行不同的代码块。Python 的语法极度依赖缩进。

### 基本结构

```python
if 条件1:
    语句块1
elif 条件2:
    语句块2
else:
    语句块3
```

- `if` 是必须的，`elif` 可以有零或多个，`else` 最多一个。
- 条件从上到下依次求值，第一个为真的分支被执行，剩下的全部跳过——即便后续条件也满足。
- 缩进严格决定代码块的边界，忘记缩进或缩进不一致会导致 `IndentationError`。

```python
score = 85
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"        # 85 在此处截住，不会到下一个
elif score >= 70:
    grade = "C"
else:
    grade = "F"
print(grade)   # B
```

### 三元表达式

Python 没有 C 系语言的三目运算符 `? :`，但有一个紧凑的条件表达式：

```python
value_if_true if condition else value_if_false
```

示例：

```python
status = "成年" if age >= 18 else "未成年"
```

它适合简单的值选择，多层嵌套会让可读性骤降，此时拆成 `if-elif` 是更好的选择。

### 缩进的纪律

!!! note "再次强调"
    使用 4 个空格缩进，不要混用 Tab。如果条件分支下只有一行语句，可以和条件写在同一行（虽语法允许，但一般不推荐）。
    ```python
    if x > 0: print("正")   # 能运行，但不美观
    ```
    保持简洁和统一永远是 Python 风格的第一要义。

## for循环与迭代协议

> 当需要对容器中的每个元素执行相同操作时，`for` 循环是主力。Python 的 `for` 循环本质上是 **遍历可迭代对象** 中的每一项，而不是像 C 语言那样通过计数器“数数”。

### 基本语法

```python
for 变量 in 可迭代对象:
    循环体
```

示例：

```python
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)
```

这里 `fruits` 是一个列表（可迭代对象），`fruit` 在每轮循环被依次赋值为列表中的元素。字符串也是可迭代对象：

```python
for char in "hello":
    print(char)   # h e l l o
```

### 迭代协议浅释

`for` 循环在背后做了三件事：

1. 调用 `iter(可迭代对象)` 获取一个迭代器。
2. 反复调用 `next(迭代器)` 获取下一个元素。
3. 当 `next()` 抛出 `StopIteration` 异常时，循环正常结束。

你不需要记住细节，只要知道一切能塞进 `for ... in ...` 的东西都是可迭代对象：列表、字符串、元组、字典、集合、文件对象、生成器等。这使得 Python 的 `for` 无比通用。

### 遍历字典

字典的 `for key in dict` 遍历的是键。要遍历值或键值对，用 `values()` 和 `items()`。

```python
d = {"x": 1, "y": 2}
for k, v in d.items():
    print(k, v)
```

### 循环的变量泄露

!!! note "注意"
    `for` 循环中定义的循环变量在循环结束后依然存在，值为最后一次迭代赋的值。这不是 bug，而是一个设计特性，不过最好别依赖它。
    ```python
    for i in range(3):
        pass
    print(i)   # 2，i 仍然可见
    ```

## range()函数与枚举enumerate()

### range() —— 整数序列生成器

`range()` 不是列表，而是一个惰性、不可变的序列对象，代表一个等差的整数范围。用在 `for` 循环中极为高效，无论范围多大，都只占极小的内存。

```python
range(stop)              # 0 到 stop-1
range(start, stop)       # start 到 stop-1
range(start, stop, step) # 步长 step，可正可负
```

!!! note "注意"
    **含头不含尾**，`range(3)` 产生 0、1、2。
    ```python
    for i in range(5):            # 0..4
        print(i)
    for i in range(2, 6):         # 2,3,4,5
        print(i)
    for i in range(10, 0, -2):    # 10,8,6,4,2
        print(i)
    ```
    如果需要立即得到一个真实列表，可以用 `list(range(5))`。

### enumerate() —— 同时拿到索引和元素

当需要知道每个元素的序号时，`enumerate()` 是标配。它返回一个 **产生 `(index, value)` 元组的可迭代对象**。

```python
fruits = ["apple", "banana", "cherry"]
for index, fruit in enumerate(fruits):
    print(f"{index}: {fruit}")
# 输出：
# 0: apple
# 1: banana
# 2: cherry
```

`enumerate()` 接受可选的 `start` 参数指定起始索引（默认 0），从 1 开始读数更符合人类习惯：

```python
for idx, fruit in enumerate(fruits, start=1):
    print(f"{idx}. {fruit}")
```

!!! tip
    在需要跟踪位置、显示编号、并行修改列表等场景，`enumerate()` 远比用 `range(len(...))` 优雅和安全。
    ```python
    # 不推荐的写法
    for i in range(len(fruits)):
        print(i, fruits[i])
    # 推荐的写法
    for i, fruit in enumerate(fruits):
        print(i, fruit)
    ```

## while循环与终止条件设计

> `while` 循环在条件为真时反复执行一段代码，适合“不知道具体循环多少次，而由某个状态决定何时停止”的场景。

### 基本语法

```python
while 条件:
    循环体
```

条件在每次循环开始前求值，若为 `True` 则执行一轮，然后再次检查。一旦条件变成 `False`，循环结束。

```python
count = 0
while count < 5:
    print(count)
    count += 1     # 千万别忘了更新条件变量！
```

### 死循环与终止设计

!!! warning "注意"
    忘记更新条件、或者条件永远为真，会导致 **死循环**。在需要持续运行的服务器或游戏循环中，死循环是刻意为之的，配合 `break` 跳出。但日常编程务必确保循环有终止路径。
    ```python
    # 刻意死循环，由内部 break 跳出
    while True:
        cmd = input("输入 'quit' 退出: ")
        if cmd == "quit":
            break
        print(f"你输入了: {cmd}")
    ```

!!! tip "while 与 for 的选择"
    - 当迭代对象已知（列表、区间等），用 `for`。
    - 当终止条件依赖某个动态状态，且迭代次数未知时，用 `while`。
    - 能用 `for` 的地方绝不用 `while` 去模拟计数器循环——更安全，也更 Pythonic。

## 循环控制

> Python 提供了三个与控制流密切相关的语句：`break`、`continue`，以及一个独特的 `else` 子句。

### break

执行 `break` 会跳出离它最近的 `for` 或 `while` 循环，循环 `else` 子句（如果有）也不会执行。

```python
for i in range(10):
    if i == 5:
        break         # 循环在第5次迭代时结束
    print(i)
# 输出：0 1 2 3 4
```

### continue

`continue` 不会终止循环，而是直接跳转到循环头部，进行下一轮迭代的条件判断。

```python
for i in range(5):
    if i % 2 == 0:
        continue      # 跳过偶数，不执行后续 print
    print(i)
# 输出：1 3
```

### * 循环的 else 子句

!!! note "注意"
    在 `for` 或 `while` 循环后面可以跟一个 `else` 块。它的含义是：**如果循环正常结束（即没有遇到 `break`），就执行 `else` 块**。被 `break` 终止的循环，`else` 部分会被跳过。
    ```python
    for i in range(5):
        if i == 100:
            break
    else:
        print("没有被打断")
    # 输出：没有被打断
    ```
    这个 `else` 最经典的用途是“查找并验证”模式：
    ```python
    def has_prime(nums):
        for n in nums:
            if is_prime(n):
                print(f"找到质数 {n}")
                break
        else:
            print("没有质数")
    ```
    如果没有预先了解，这个 `else` 的语义很容易误解为“循环结束后必然执行”。建议一开始觉得别扭就写注释，看多了会习惯它的妙处。

## 嵌套循环与典型算法实例

> 组合多种控制结构解决实际问题，是编程能力的分水岭。嵌套循环就是在一个循环体内再放另一个循环，常用来处理多维数据、排列组合和经典算法。

### 嵌套循环的执行过程

外层循环每执行一次，内层循环会完整跑完一圈。

```python
for i in range(3):
    for j in range(2):
        print(f"({i}, {j})")
# 输出：(0,0) (0,1) (1,0) (1,1) (2,0) (2,1)
```

### 实例一：打印九九乘法表

```python
for i in range(1, 10):
    for j in range(1, i+1):
        print(f"{j}×{i}={i*j:<2}", end="  ")
    print()   # 每行结束换行
```

这里内层循环范围依赖外层循环变量 `i`，利用字符串格式化对齐列，最终打印出熟悉的三角形乘法表。

### 实例二：冒泡排序（基础排序算法）

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n-1):
        swapped = False
        for j in range(n-1-i):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
                swapped = True
        if not swapped:   # 若本轮未发生交换，说明已有序
            break
    return arr

nums = [64, 34, 25, 12, 22, 11, 90]
print(bubble_sort(nums))  # [11, 12, 22, 25, 34, 64, 90]
```

这个例子融合了嵌套循环、条件判断、`break` 优化、多变量交换等知识，是前面几节内容的综合应用。

### 实例三：寻找列表中所有和为 0 的不重复数对

```python
nums = [3, 4, -7, 1, -1, 6, -3]
pairs = []
for i in range(len(nums)):
    for j in range(i+1, len(nums)):
        if nums[i] + nums[j] == 0:
            pair = sorted((nums[i], nums[j]))  # 标准化顺序
            if pair not in pairs:
                pairs.append(pair)
print(pairs)   # [[-3, 3], [-1, 1], [-4, 4]]  (若包含4)
```

这里利用内层循环从 `i+1` 开始避免重复比较，同时用排序和成员检查保证返回结果唯一。虽然大 O 复杂度不理想（数据量大会慢），但它清晰地展示了嵌套循环 + 条件的组合套路。

## 本章小结

现在你的程序已经会“思考”了。布尔类型和比较运算符提供了判断的原子能力，逻辑运算符让你能拼接复杂条件。`if-elif-else` 为程序引入了分支决策，`for` 和 `while` 则让它可以重复劳动。`range` 和 `enumerate` 是标准化计数的左膀右臂，`break`、`continue`、`else` 精细调控循环的流向。嵌套循环把这些元素组合起来，解决真正的算法问题。下一章，我们将踏入函数的殿堂——把代码组织成可复用的模块，告别复制粘贴的蛮荒时代。
