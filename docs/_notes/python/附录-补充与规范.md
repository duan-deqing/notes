# 附录 补充与规范

## A. PEP 8 编码风格完整指南

[PEP 8](https://peps.python.org/pep-0008/) 是 Python 社区公认的编码风格指南，由 Guido van Rossum 和 Barry Warsaw 合写。遵循它能让你的代码与整个生态保持一致的“面孔”，提升可读性与可维护性。下面是你需要穷尽一生的要点摘录，但它并不死板——可读性永远是最高的裁决标准。

### 代码布局

- **缩进**：每级缩进使用 **4 个空格**。绝不混用制表符和空格；设置编辑器为“按 Tab 键插入空格”。
- **每行最大长度**：所有行限制在 79 个字符；文档字符串或注释限制在 72 个字符。续行可以悬挂缩进，但应与首行内容区分。
- **空行**：顶层函数和类定义之间用两个空行；类内部的方法定义之间用一个空行。在逻辑段落之间也可酌情加入空行。
- **导入**：
  - 每个导入独占一行；`import os, sys` 这种写法不推荐。
  - 导入顺序：`标准库 -> 第三方库 -> 本地模块`，每组之间空一行。
  - 尽量使用绝对导入，相对导入仅限于包内部。
  - 避免使用 `from module import *`，它会污染命名空间。

### 空格

- 逗号、分号、冒号后加一个空格，前不加。
- 函数调用时，函数名和括号间不加空格：`print(x)`，而非 `print (x)`。
- 索引或切片时，冒号两侧不加空格。
- 赋值等二元运算符两侧加一个空格（`a = b + c`），但函数参数中默认值的 `=` 两侧不加（`def func(param=0):`）。
- 如果使用具有不同优先级的运算符，可以考虑在最低优先级的运算符周围添加空格。

### 命名约定

| 类型                     | 命名规则                     | 示例                           |
| :----------------------- | :--------------------------- | :----------------------------- |
| 变量、函数、方法         | `snake_case`                 | `user_name`, `calculate_tax()` |
| 常量                     | `UPPER_CASE`                 | `MAX_SIZE`, `DEFAULT_PORT`     |
| 类                       | `UpperCamelCase`             | `DataLoader`, `HttpClient`     |
| 模块和包                 | 简短的全小写名字，可用下划线 | `utils.py`, `my_package`       |
| 私有成员                 | 单个下划线前缀               | `_internal_cache`              |
| 避免名字冲突             | 双下划线前缀（会被名称改写） | `__id`                         |
| 避免与 Python 关键字冲突 | 尾部加单下划线               | `class_`, `list_`              |
| 特殊方法                 | 双下划线包裹                 | `__init__`, `__str__`          |

### 注释与文档字符串

- 注释应该解释 **为什么** 这么做，而不是做什么（代码本身应足够清晰）。
- 块注释每行以 `#` 加一个空格开头，与所在代码对齐；行内注释应与语句至少间隔两个空格。
- 文档字符串（docstring）：用三重引号，放在模块、函数、类或方法顶部的首行。一句话总结，之后可展开。对于函数，推荐描述参数、返回值、可能抛出的异常。

!!! tip "编程建议"
    - 比较 `None` 时用 `is` 或 `is not`，而非 `==`。
    - 不要与 `True` / `False` 显式比较：`if x:` 而不是 `if x == True:`。
    - 序列或映射不为空时，直接用真值测试：`if items:`。
    - 优先使用字符串的 `.join()` 拼接，而非 `+` 在循环里反复构建。

风格指南服务于人与人的沟通，而不是机器的强迫症。当规则让代码更难读时，可以打破它——但要明确记下原因。

## B. 常用内置函数分类速查

> Python 内置函数约 70 个，此处按功能分组，列出最高频的成员并附简短说明。它们是处理日常任务的“瑞士军刀”。

### 类型转换与构造

| 函数          | 说明                              |
| :------------ | :-------------------------------- |
| `int(x)`      | 转换为整数；可传入字符串和基数    |
| `float(x)`    | 转换为浮点数                      |
| `str(x)`      | 转换为字符串                      |
| `bool(x)`     | 转换为布尔值；空/零/None 为 False |
| `list(iter)`  | 从可迭代对象创建列表              |
| `tuple(iter)` | 创建元组                          |
| `set(iter)`   | 创建集合                          |
| `dict(**kw)`  | 从关键字参数或可迭代对象创建字典  |

### 数学与聚合

| 函数                      | 说明                                            |
| :------------------------ | :---------------------------------------------- |
| `abs(x)`                  | 绝对值                                          |
| `round(x, n)`             | 四舍五入；注意浮点舍入策略（银行家舍入）        |
| `pow(x, y, mod)`          | 返回 `x**y`，若提供 `mod` 则返回 `(x**y) % mod` |
| `sum(iter, start)`        | 求和                                            |
| `min(iter)` / `max(iter)` | 最小/最大值                                     |
| `divmod(a, b)`            | 返回 `(a//b, a%b)` 元组                         |

### 迭代与循环辅助

| 函数                           | 说明                             |
| :----------------------------- | :------------------------------- |
| `len(obj)`                     | 返回容器长度                     |
| `range([start], stop, [step])` | 生成不可变整数序列               |
| `enumerate(iter, start=0)`     | 产生 `(index, value)` 对         |
| `zip(*iters)`                  | 并行迭代，返回元组迭代器         |
| `reversed(seq)`                | 返回反向迭代器                   |
| `sorted(iter, key, reverse)`   | 返回新排序列表，原数据不变       |
| `filter(func, iter)`           | 过滤，保留 func 返回 True 的元素 |
| `map(func, iter)`              | 对每个元素应用 func，返回迭代器  |
| `all(iter)` / `any(iter)`      | 全部为真？/ 任一为真？           |

### 对象与属性

| 函数                          | 说明                         |
| :---------------------------- | :--------------------------- |
| `type(obj)`                   | 返回对象的类                 |
| `isinstance(obj, class)`      | 类型检查（考虑继承）         |
| `hasattr(obj, name)`          | 是否有某属性                 |
| `getattr(obj, name, default)` | 安全获取属性                 |
| `setattr(obj, name, value)`   | 设置属性                     |
| `dir(obj)`                    | 返回属性和方法列表           |
| `id(obj)`                     | 返回对象唯一标识（内存地址） |
| `repr(obj)`                   | 返回官方字符串表示           |
| `str(obj)`                    | 返回用户友好的字符串表示     |

### 输入输出与文件

| 函数                           | 说明           |
| :----------------------------- | :------------- |
| `print(*objs, sep, end, file)` | 输出           |
| `input(prompt)`                | 读取一行字符串 |
| `open(file, mode, encoding)`   | 打开文件       |

### 高阶函数相关

| 函数                                | 说明                      |
| :---------------------------------- | :------------------------ |
| `sorted(iter, key, reverse)`        | 自定义排序                |
| `max(iter, key)` / `min(iter, key)` | 带键函数的最大最小值      |
| `iter(obj)` / `next(it, default)`   | 获取迭代器/获取下一个元素 |

!!! note "注意"
    这并非完整清单（例如 `eval`、`exec` 慎用；`help`、`vars` 等辅助函数未列），但包含了 95% 日常编程所需的内置函数。当你需要某个功能时，先查一下内置函数列表，很可能轮子已经造好了。

## C. 标准库模块精选简介

> Python 标准库是“自带电池”的典范。在基础内容中学过 `math`、`random`、`datetime`、`json`、`csv` 等，这里补充一些同样常用、但教程正文未深入展开的模块，帮你建立一份“能力地图”。

### 系统与运行环境

- **`sys`**：解释器交互。常用属性：`sys.argv`（命令行参数）、`sys.path`（模块搜索路径）、`sys.exit()`、`sys.version`、`sys.platform`。
- **`os`**：操作系统接口。文件/目录操作（`os.remove`, `os.makedirs`）、环境变量（`os.environ`）、进程管理。许多操作可用 `pathlib` 替代。
- **`pathlib`**：面向对象的文件系统路径操作。`Path.home() / 'folder' / 'file.txt'` 比字符串拼接直观安全。
- **`subprocess`**：执行外部命令，比 `os.system` 更强大安全，推荐用于脚本自动化。

### 数据处理与序列化

- **`csv`**：读写 CSV 文件，处理方言与编码。
- **`json`**：JSON 序列化与反序列化。
- **`pickle`**：Python 对象序列化，可将任意 Python 对象存入文件（不可跨语言，注意安全）。
- **`base64`**：Base64 编解码，常用于二进制数据文本化。

### 文本处理

- **`re`**：正则表达式，强大的模式匹配、搜索替换。需要时查阅速查表。
- **`string`**：字符串常量和辅助类（如 `string.ascii_letters`、`Formatter`）。
- **`textwrap`**：格式化文本段落、缩进、自动换行。
- **`difflib`**：文本差异比较。

### 数据结构扩展

- **`collections`**：
  - `namedtuple()`：可命名的元组，轻量数据对象。
  - `deque`：双端队列，两端 `append`/`pop` 均为 O(1)。
  - `Counter`：计数器，统计可迭代对象的频次。
  - `defaultdict`：带默认值的字典，避免 KeyError。
  - `OrderedDict`：有序字典（Python 3.7+ 内置 `dict` 已保序，但仍有用）。
- **`heapq`**：堆队列算法（优先队列）。
- **`bisect`**：二分查找与插入，保持有序列表。

### 迭代工具

- **`itertools`**：高效迭代器构建块（第十一章已详述）。
- **`functools`**：高阶函数支持：`reduce`、`lru_cache`（缓存）、`wraps`（装饰器）、`partial`（偏函数）。

### 时间与日期

- **`time`**：时间访问（时间戳、睡眠 `sleep`）。
- **`datetime`**：日期、时间、时间差的计算与格式化。

### 并发与异步

- **`threading`**：多线程编程（受 GIL 限制，适合 IO 密集型）。
- **`multiprocessing`**：多进程，绕过 GIL 用于 CPU 密集型。
- **`asyncio`**：异步 IO 框架，编写高并发网络应用。

### 开发工具与调试

- **`logging`**：比 `print` 更专业的日志系统，支持级别、输出目标、格式。
- **`unittest`**：单元测试框架（另有流行的 `pytest` 第三方库）。
- **`pdb`**：Python 调试器，可交互式检查程序。
- **`traceback`**：提取、格式化和打印异常堆栈信息。

### 网络与互联网

- **`urllib.request`**：打开 URL，发送 HTTP 请求（第三方 `requests` 更友好）。
- **`socket`**：底层网络接口。
- **`email`**：处理电子邮件消息。
- **`http.server`**：简单的 HTTP 服务器。

### 其他实用模块

- **`argparse`**：命令行参数解析，构建用户友好的 CLI 工具。
- **`configparser`**：解析 `.ini` 配置文件。
- **`shutil`**：高级文件操作（复制、移动、压缩）。
- **`tempfile`**：生成临时文件和目录。

这仅仅是标准库的冰山一角。遇到需求时，先搜索“Python stdlib + 关键词”，几乎总有一个模块能帮你少写一半代码。

## D. 后续学习路径推荐

> 恭喜你走完了 Python 基础教程的全部核心内容。现在你已具备阅读大部分 Python 代码的能力，并能编写中等规模的实用程序。下面的路线图帮助你根据兴趣选择下一步的成长方向，每一步都有明确的里程碑。

### 1. 夯实基础 —— “把根扎深”

- **完成小项目**：用标准库写几个完整的工具，例如：文件批量重命名器、Markdown 转换器、基于 CSV 的数据分析报告生成器。项目代码量在 200~500 行即可。
- **阅读优秀代码**：在 GitHub 上找一些星标较多的纯 Python 项目（如 `requests` 早期版本），阅读源码，体会项目结构和编码风格。
- **掌握测试**：学习 `unittest` 或 `pytest`，为你的函数和类编写测试用例，养成测试驱动的习惯。

### 2. 软件工程实践 —— “从脚本到工程”

- **版本控制**：熟练使用 Git，理解分支、合并、Pull Request。
- **项目管理工具**：学习 `poetry` 或 `pipenv` 进行依赖管理，了解 `pyproject.toml`。
- **代码质量**：使用 `flake8`、`pylint` 或 `ruff` 进行静态检查；用 `black` 自动格式化代码；使用 `pre-commit` 在提交前自动执行检查。
- **设计模式**：阅读《Python 设计模式》或 Head First 设计模式，理解单例、工厂、观察者等在 Python 中的实现。

### 3. 领域方向 —— “选择一个山头”

Python 的应用极其广泛，选择一条主线深耕：

| 方向             | 核心技术栈                                             | 阶段目标                            |
| :--------------- | :----------------------------------------------------- | :---------------------------------- |
| **Web 开发**     | Django / FastAPI + 数据库（PostgreSQL, Redis）+ Docker | 搭建全栈网站或 RESTful API          |
| **数据科学**     | NumPy, pandas, matplotlib, seaborn, Jupyter            | 清洗分析真实数据集，绘制美观图表    |
| **机器学习/AI**  | scikit-learn, PyTorch/TensorFlow, Hugging Face         | 训练分类/回归模型，部署简单推理服务 |
| **自动化与脚本** | Selenium, requests, openpyxl, PyAutoGUI                | 编写爬虫、自动报表、日常任务自动化  |
| **桌面应用**     | PyQt6 / PySide6 或 Tkinter                             | 开发带图形界面的跨平台工具          |

### 4. 进阶语言特性与应用

- **异步编程**：`asyncio`、`aiohttp`，理解事件循环和协程，处理高并发 IO。
- **并发与并行**：深入 `threading`、`multiprocessing`，了解 GIL 及规避策略。
- **性能优化**：使用 `cProfile` 和 `line_profiler` 找出瓶颈；了解 `__slots__`、生成器优化；必要时学习 Cython 或 Numba。
- **类型提示**：使用 `typing` 模块，结合 `mypy` 进行静态类型检查，提升大型项目的可靠性。
- **C 扩展与嵌入**：了解如何用 C 编写 Python 模块（适合对底层感兴趣）。

### 5. 参与开源

- **贡献文档或修复小 bug**：从 good first issue 开始，体验协作流程。
- **发布自己写的包**：将一个实用工具打包并发布到 PyPI。

### 推荐资源

- **书籍**：《流畅的 Python》（深入理解语言特性）、《Python 编程：从入门到实践》（项目驱动）、《Effective Python》（最佳实践）。
- **在线学习**：Real Python、PyCon 演讲视频（许多干货）、官方的 Python 教程与 HOWTO 文档。
- **社区**：Stack Overflow、Python 中文社区、Python Discord、GitHub Discussions。

编程是一场无限游戏。基础教程只是你旅途中的第一张地图，真正的魔法都藏在持续实践中。保持好奇心，祝你写出能改变周围世界的代码。
