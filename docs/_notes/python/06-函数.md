# 函数

## 函数定义与调用栈模型

> 函数是组织代码、消除重复的基本单位。它把一段逻辑打包，取名，之后你可以反复调用而不必重写。Python 用 `def` 关键字定义函数，简洁到几乎没有仪式感。

### 定义函数

```python
def 函数名(参数列表):
    """文档字符串（可选）"""
    函数体
    return 返回值  # 可选
```

最简单的例子：

```python
def greet():
    print("你好！")
```

调用它只需函数名加括号：`greet()`。

### 文档字符串

紧跟在函数头之后的字符串（三引号）是 docstring，用于说明函数的功能。它可以通过 `help(函数名)` 或 `函数名.__doc__` 查看，是自说明代码的第一推手。

```python
def add(a, b):
    """返回 a 与 b 的和。"""
    return a + b
```

### 调用栈模型

当程序调用函数时，解释器会创建一个 **栈帧（frame）**，记录该函数的局部变量、参数和返回地址，并将其压入调用栈。函数返回时，栈帧弹出，控制权交还给调用处。这种“后进先出”的结构保证了嵌套调用的正常返回。

递归过深或误写无限递归会引发 `RecursionError`，因为调用栈有深度限制（默认约 1000 层）。这也是为什么理解栈的存在很重要。不过大多数日常函数的栈帧生命周期极短，你甚至感知不到。

### 函数是对象

在 Python 中，函数本身是对象，可以赋值给变量、作为参数传递、甚至作为返回值——这一特性直接奠定了高阶函数和装饰器的基础。

```python
def hello():
    return "Hello"

say = hello          # 将函数对象赋给变量 say
print(say())         # "Hello"
print(type(say))     # <class 'function'>
```

## 参数传递

> 参数让函数得以处理不同的数据。Python 提供了灵活多样的参数传递机制，从容应对各种调用场景。

### 位置参数

最传统的方式，按定义顺序一一传入。

```python
def power(base, exp):
    return base ** exp

result = power(2, 3)   # base=2, exp=3
```

传入的参数个数必须与定义严格匹配，少了或多了都会引发 `TypeError`。

### 默认参数

为参数提供默认值，调用时可以省略。带有默认值的参数必须放在所有无默认值参数的 **后面**。

```python
def power(base, exp=2):
    return base ** exp

print(power(3))      # 9，exp 使用默认值 2
print(power(3, 3))   # 27
```

!!! warning "**关键陷阱**"
    默认参数的默认值在函数 **定义时** 只计算一次，之后每次调用都共享同一个对象。如果默认值是可变对象（如列表、字典），会出现意料之外的“记忆效应”。
    ```python
    def append_to(element, target=[]):
        target.append(element)
        return target

    print(append_to(1))   # [1]
    print(append_to(2))   # [1, 2] —— target 是同一个列表！
    ```

    正确做法是使用 `None` 作为默认值，内部再初始化：
    ```python
    def append_to(element, target=None):
        if target is None:
            target = []
        target.append(element)
        return target
    ```

### 关键字参数

调用时，可以用 `key=value` 的形式指定参数，此时顺序无关紧要。

```python
def describe_pet(name, animal_type):
    print(f"{name} 是一只 {animal_type}")

describe_pet(animal_type="狗", name="旺财")
```

关键字参数能大幅提升可读性，建议对于含义模糊的参数（布尔值、数字）总是用关键字调用。也可以混用位置参数和关键字参数，但 **位置参数必须在关键字参数之前**。

```python
describe_pet("旺财", animal_type="狗")  # 合法
describe_pet(name="旺财", "狗")         # 语法错误
```

## 可变参数：*args 与 **kwargs

> 当函数需要接收不确定数量的参数时，`*args` 和 `**kwargs` 登场了。名字 `args` 和 `kwargs` 只是习惯，关键在星号。

### *args —— 打包多余位置参数

`*args` 会将所有剩余的位置参数收集到一个 **元组** 中。

```python
def multiply(*numbers):
    result = 1
    for n in numbers:
        result *= n
    return result

print(multiply(2, 3, 4))   # 24
print(multiply())           # 1 （空元组）
```

可以在普通参数之后使用 `*args` 收集额外参数：

```python
def log(level, *messages):
    for msg in messages:
        print(f"[{level}] {msg}")

log("ERROR", "连接超时", "重试失败")
# [ERROR] 连接超时
# [ERROR] 重试失败
```

### **kwargs —— 打包多余关键字参数

`**kwargs` 收集多余的关键字参数，打包成一个 **字典**。

```python
def create_profile(**details):
    for key, value in details.items():
        print(f"{key}: {value}")

create_profile(name="Ada", age=25, city="Hangzhou")
```

结合位置参数、`*args`、`**kwargs`，定义一个“全功能签名”的顺序必须是：

1. 普通位置参数
2. `*args`
3. 带默认值的参数（或仅限关键字参数）
4. `**kwargs`

例如：

```python
def func(a, b, *args, c=10, **kwargs):
    print(a, b, args, c, kwargs)

func(1, 2, 3, 4, c=99, x=100, y=200)
# 1 2 (3, 4) 99 {'x': 100, 'y': 200}
```

### 解包参数

星号也可以反过来用在调用端，将序列或字典“打散”为独立参数。

```python
def add(a, b, c):
    return a + b + c

values = [1, 2, 3]
print(add(*values))          # 等价于 add(1,2,3)

data = {"a": 1, "b": 2, "c": 3}
print(add(**data))           # 等价于 add(a=1,b=2,c=3)
```

这种“打包—解包”的对称性让编写通用函数转发器（wrapper）极其顺畅。

## return语句与多返回值

> 函数通过 `return` 语句将计算结果传回调用处。

### return 的基本规则

- 遇到 `return` 函数立即终止，后面的代码不会执行。
- 没有 `return` 或 `return` 后面不跟值时，函数默认返回 `None`。
- 可以有多个 `return`，通常分布在条件分支中。

```python
def absolute(x):
    if x < 0:
        return -x
    return x
```

### 返回多个值

Python 函数可以“返回多个值”，实际上是把它们打包进一个 **元组**，然后利用解包在调用侧拆开。

```python
def min_max(items):
    return min(items), max(items)    # 返回元组

low, high = min_max([3, 1, 4, 1, 5])
print(low, high)   # 1 5
```

如果只用单个变量接收，就会得到一个完整的元组：

```py
result = min_max([3,1,4,1,5])
print(result)   # (1, 5)
```

从 Python 3.8 开始还可使用“分配表达式”（海象运算符）配合，但多返回值主要靠元组包装，简单而可靠。

### 尽早返回原则

为了让代码可读性更好，推荐“尽早返回”的风格：在函数开头检查特殊条件并直接返回，避免深层嵌套的 `if...else`。

```python
def get_score(student):
    if student is None:
        return 0
    if student.score is None:
        return 0
    return student.score
```

## 作用域规则与LEGB查找顺序

> 变量在程序的不同位置有不同的“可见性”，这套规则叫作用域。Python 的作用域由 **LEGB** 四个层级构成，查找变量时按此顺序：

| 字母 | 含义      | 说明                                   |
| :--- | :-------- | :------------------------------------- |
| L    | Local     | 函数内部定义的局部变量                 |
| E    | Enclosing | 外层函数的局部作用域（闭包场景）       |
| G    | Global    | 模块级的全局变量                       |
| B    | Built-in  | Python 内置的名字（如 `print`、`len`） |

### 局部作用域

在函数内赋值的变量默认是局部的，对外部是不可见的。

```python
def func():
    x = 100   # 局部变量
    print(x)

func()
print(x)   # NameError: name 'x' is not defined
```

### 嵌套函数与闭包

内部函数可以读取外部函数的变量（非全局），但如果要 **修改** 它，必须用 `nonlocal` 声明。

```python
def outer():
    count = 0
    def inner():
        nonlocal count
        count += 1
        return count
    return inner

counter = outer()
print(counter())   # 1
print(counter())   # 2
```

### 全局作用域

在模块顶层定义的变量是全局的。函数内若要修改全局变量，需使用 `global` 关键字。

```python
total = 0

def add(n):
    global total
    total += n

add(5)
print(total)   # 5
```

### 慎用 global

频繁读写全局变量会增加代码耦合度，测试和调试都困难。尽量用参数和返回值传递数据，确实需要模块级状态时，也推荐将其封装在类中而非裸露的全局变量。

### LEGB 实践

遇到变量时，解释器由内向外查找。理解这个顺序可以避免因变量遮蔽而引发鬼畜 bug。举例来说，若在函数内定义了同名变量，它就会覆盖外部变量，你得到的将始终是局部版本。

## 匿名函数：lambda表达式

Python 使用 `lambda` 关键字创建一次性小函数，它的语法极其精简：

```python
lambda 参数: 表达式
```

- 只能有一个表达式，不能包含语句（如 `print`、赋值、循环）。
- 计算结果会自动作为返回值，不需要 `return`。
- 可以用在需要函数对象作为参数的地方，如 `sorted` 的 `key` 参数。

### 常见场景

- **排序或 max/min 的键函数**

```python
words = ["apple", "banana", "cherry"]
longest = max(words, key=lambda w: len(w))
print(longest)   # "banana"
```

- **配合 map、filter**

```python
nums = [1, 2, 3, 4]
squares = list(map(lambda x: x**2, nums))   # [1, 4, 9, 16]
evens = list(filter(lambda x: x % 2 == 0, nums))  # [2, 4]
```

lambda 还可以立即调用（虽然很少这么做）：

```python
result = (lambda x, y: x + y)(3, 4)   # 7
```

!!! note "lambda 的界限"
    如果一个逻辑需要用多行来表达，或有循环、异常处理等，就老老实实定义一个 `def` 函数，不要强行塞进 lambda。可读性远比少写两行代码重要。

## 递归函数设计

> 递归就是函数自己调用自己，是解决具有自相似结构问题（如树遍历、分治算法）的优雅方式。Python 递归受限于调用栈深度，但正确编写仍清晰有力。

### 递归的两个要素

1. **基线条件**：告诉函数何时停止，防止无限循环。
2. **递归条件**：将问题分解为更小的子问题，向基线条件靠近。

计算阶乘是最简单的例子：

```python
def factorial(n):
    if n == 1:          # 基线
        return 1
    return n * factorial(n - 1)   # 递归
```

### 尾递归与优化限制

如果一个递归调用是函数最后执行的动作，它就叫尾递归。某些语言会对尾递归进行优化，但 **Python 解释器目前不会**，因此递归过深依然会爆栈。可以用 `sys.setrecursionlimit` 提高限制，但治标不治本。大规模递归最好转用循环或借助显式栈。

### 典型应用：斐波那契数列

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

简单却效率极低（重复计算多），引入缓存（记忆化）可以脱胎换骨：

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

这个例子也预告了装饰器的神奇作用。

## 高阶函数

> 函数可以作为参数传入另一个函数，也可以作为返回值，这类操作其他函数的函数称为 **高阶函数**。Python 内置了几个十分经典的高阶函数。

### map(function, iterable, ...)

对可迭代对象中的每一个元素调用函数，返回一个迭代器，产生结果序列。

```python
nums = [1, 2, 3, 4]
doubled = list(map(lambda x: x * 2, nums))   # [2, 4, 6, 8]
```

可以传入多个可迭代对象，对应多参数函数：

```python
a = [1, 2, 3]
b = [10, 20, 30]
summed = list(map(lambda x, y: x + y, a, b))   # [11, 22, 33]
```

### filter(function, iterable)

调用函数过滤：保留函数返回 `True` 的元素。当函数为 `None` 时，直接滤掉所有假值。

```python
nums = [0, 1, 2, 3, None, 4]
truthy = list(filter(None, nums))          # [1, 2, 3, 4]
evens = list(filter(lambda x: x % 2 == 0, range(10)))  # [0, 2, 4, 6, 8]
```

### reduce(function, iterable[, initial])

`reduce()` 位于 `functools` 模块，它将函数反复应用于序列，累积出单一结果。

```python
from functools import reduce

sum_all = reduce(lambda x, y: x + y, [1, 2, 3, 4], 0)   # 10
# 内部过程：((((0+1)+2)+3)+4)
```

可以用它实现阶乘、字符串拼接等任何可归结为二元累积的操作。

### 推导式常是更优选择

Python 的列表推导式和生成器表达式在许多场景下比 `map`/`filter` 更直观。比如 `[x*2 for x in nums]` 比 `list(map(lambda x: x*2, nums))` 更 Pythonic。选择的标准是 **可读性优先**，当操作简单且有现成函数时（如 `map(str, data)`），高阶函数仍然简洁。

## 装饰器基础与@语法糖

> 装饰器（Decorator）本质是 **接受一个函数，返回一个新函数** 的可调用对象。它让你可以在不修改原函数代码的情况下，为函数附加额外行为（日志、计时、权限检查等）。`@` 符号只是语法糖，使代码更美。

### 最简单的装饰器

```python
def my_decorator(func):
    def wrapper():
        print("函数执行前")
        func()
        print("函数执行后")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
# 输出：
# 函数执行前
# Hello!
# 函数执行后
```

`@my_decorator` 相当于 `say_hello = my_decorator(say_hello)`，重新绑定了标识符。

### 处理被装饰函数的参数

为了让装饰器通用，内部 `wrapper` 需用 `*args, **kwargs` 接受任意参数，并原样传递。

```python
def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f"{func.__name__} 耗时 {end - start:.4f}s")
        return result
    return wrapper

@timer
def compute(n):
    return sum(range(n))

print(compute(1000000))
```

### 保留元信息

直接装饰会丢失原函数的 `__name__`、`__doc__` 等元数据。解决方式是在 `wrapper` 上使用 `functools.wraps`。

```python
from functools import wraps

def logger(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}")
        return func(*args, **kwargs)
    return wrapper
```

### 带参数的装饰器

如果需要向装饰器本身传入参数（如指定重试次数），就需要再多一层嵌套——实际上装饰器工厂返回一个真正的装饰器。

```python
def retry(times):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for i in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if i == times - 1:
                        raise
            return None
        return wrapper
    return decorator

@retry(3)
def risky_operation():
    # ...
```

装饰器是 Python 中“面向切面编程”的基石，熟练后会让你的代码保持单一职责，同时灵活扩展。

## 本章小结

函数是编程中的“动词”，是你开始构建可靠、可复用软件的核心。从基本的 `def` 和参数传递，到 *args/**kwargs 的灵活变长，再到作用域 LEGB 规则，接着是 lambda、递归和高阶函数，最后以装饰器收尾——你已经掌握了 Python 函数编程的整套思维工具。接下来，我们将进入模块与包的领域，学习如何把函数和类组织成更大规模的项目结构。
