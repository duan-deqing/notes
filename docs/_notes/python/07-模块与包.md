# 模块与包

## 模块导入机制：import与from...import

当你需要复用代码时，模块是第一层组织单位。一个 `.py` 文件就是一个模块，文件名（不含扩展名）即模块名。把相关的函数、类和数据定义放在同一个模块里，然后用 `import` 将其引入当前命名空间，这是 Python 模块化编程的起点。

### import 语句的基本形式

**import module** 形式会将整个模块放到一个独立命名空间中，防止名称污染。使用模块内的名字时必须加上模块名作为前缀。

```python
import math

print(math.sqrt(16))   # 4.0
print(math.pi)         # 3.141592653589793
```

**给模块起别名**：当模块名较长或存在命名冲突时，用 `as` 指定别名。

```python
import numpy as np
import matplotlib.pyplot as plt
```

**一次导入多个模块**：可以用逗号分隔，但 PEP 8 推荐每个导入独占一行。

```python
import os
import sys
import json
```

### from ... import ... 形式

这种形式直接把模块中指定的名字注入当前命名空间，使用时不再需要模块前缀，但要小心与已有名字冲突。

```python
from math import sqrt, pi

print(sqrt(25))   # 5.0
print(pi)         # 3.141592653589793
```

!!! note "**导入全部名字（不推荐）**"
    `from module import *` 会把模块内所有不以单下划线开头的公开名字一股脑拉进来，很容易造成命名污染和可读性下降，仅适合在交互式环境或极少数特殊场景使用。
    可以导入后重命名避免冲突：
    ```python
    from math import sqrt as square_root
    ```

### 导入原理简述

执行 `import` 时，解释器会经历以下步骤：

1. **搜索**：在 `sys.path` 定义的目录列表中寻找对应模块的 `.py` 文件或动态库。
2. **编译**：将找到的源码编译为字节码（如果 `.pyc` 文件过时或不存在则重新生成）。
3. **执行**：从模块顶部到底部执行一遍模块代码，在运行时绑定所有定义。
4. **缓存**：将模块对象存入 `sys.modules`。重复 `import` 同一个模块时，解释器直接从缓存返回对象，**不会重复执行模块代码**。

因此，即使你多次导入相同模块，也只会执行一次顶层代码。这个缓存特性是性能优化，也决定了一些初始化操作只需要做一次。

### 相对导入（包内使用）

在包内部，一个模块可以相对路径导入同包的兄弟模块，语法是 `.`（当前包）和 `..`（上级包）。

```python
from . import sibling_module
from .. import parent_module
```

相对导入只能在包内使用，直接在顶层脚本中执行相对导入会报错。日常基础开发中，可以先熟悉绝对导入，再接触相对导入。

## 模块搜索路径与命名空间管理

### sys.path —— 模块搜索的路标

当你写下 `import foo` 时，Python 依次在 `sys.path` 的每个目录中查找 `foo.py`。`sys.path` 是一个字符串列表，它的初始化顺序如下：

1. 当前脚本所在目录（或空字符串表示当前工作目录）。
2. `PYTHONPATH` 环境变量中列出的目录。
3. Python 安装目录下的标准库路径。
4. 第三方包安装的 `site-packages` 目录。

可以在运行时查看或修改 `sys.path`（但通常不推荐随意更改）。

```python
import sys
print(sys.path)
```

!!! note "临时添加搜素路径"
    如果想临时增加搜索路径，可以：
    ```python
    sys.path.insert(0, "/path/to/my/modules")
    ```
    放置在列表较前位置会提升查找优先权。

### 命名空间管理

每个模块拥有自己的 **全局作用域**。在 `import` 一个模块之后，被导入的名字会挂载到模块对象上，构成以模块名为前缀的隔离区域。

```python
import math
print(dir(math))   # 查看 math 模块里的所有名字
```

用 `from ... import` 直接拉进来的名字会挂在当前模块的全局作用域，你可能无意间遮蔽内置名字（如 `from math import sin` 之后前面定义的 `sin` 会被覆盖）。为此，大型项目偏向使用 `import module` 形式再加别名。

### 重新加载模块

开发过程中修改了某个模块，想在不退出解释器的情况下重新加载，可以使用 `importlib.reload`。但 `reload` 有局限性，例如不会更新旧的实例引用等。日常开发更推荐重启解释器或 Jupyter 内核。

```python
import importlib
importlib.reload(module)
```

## 编写自定义模块与`__name__`守护

### 创建自定义模块

假设你写了一个名为 `string_utils.py` 的文件，里面放两个函数：

```python
# string_utils.py
def capitalize_words(text):
    """将每个单词的首字母大写"""
    return ' '.join(word.capitalize() for word in text.split())

def remove_duplicate_spaces(text):
    """将连续的多个空格替换为单个空格"""
    return ' '.join(text.split())
```

在同一目录下的另一个脚本 `main.py` 中，你可以这样使用它：

```python
import string_utils

clean = string_utils.remove_duplicate_spaces("Hello     world")
print(string_utils.capitalize_words(clean))   # Hello World
```

### `__name__` 与执行守护

每个 Python 模块都有一个内置属性 `__name__`。

- 当模块被直接作为脚本运行时，`__name__` 的值是 `"__main__"`。
- 当模块被 `import` 导入时，`__name__` 的值是模块本身的 **文件名**（不带 `.py`）。

利用这一特性，可以在模块底部放一个条件，使得某些代码 **只在该文件直接运行时才执行，而被导入时不执行**。

```python
# string_utils.py
def main():
    print("这是一个演示")

if __name__ == "__main__":
    main()
```

这让你既能将 `string_utils.py` 当作一个工具模块复用，又能独立运行它进行自测。这是 Python 社区中极为通用的模式。

## * 包的构建与`__init__.py`的作用

当一个功能域涉及多个模块时，将它们组织成 **包（package）** 是必然选择。包本质就是一个包含 `__init__.py` 文件的目录。这个文件告诉解释器：这个目录应当被视为一个 Python 包，而不是普通文件夹。

### 包的目录结构示例

```text
mypackage/
├── __init__.py
├── core.py
├── helpers.py
└── subpackage/
    ├── __init__.py
    └── plugin.py
```

### `__init__.py` 的三种角色

1. **标记与初始化**：
    哪怕内容为空，该文件也是必需的（Python 3.3 引入了“隐式命名空间包”，但在实践中有 `__init__.py` 仍是最清晰的做法）。当包被首次导入时，`__init__.py` 会被执行，你可以在这里放置包的初始化逻辑，或暴露包级别的公共 API。

2. **暴露接口**：
    在 `__init__.py` 里通过 `from .module import ClassA` 把内部模块中的名字提升到包层面，让下游用户能用 `mypackage.ClassA` 这样简洁的方式访问。

    ```python
    # __init__.py
    from .core import Database, Cache
    from .helpers import format_date
    ```

3. **定义 `__all__`**：
   如果希望控制 `from mypackage import *` 时暴露哪些名字，可以在 `__init__.py` 中定义一个字符串列表 `__all__`。

    ```python
    __all__ = ["Database", "Cache", "format_date"]
    ```

### 包内导入

!!! tip
    在包内部的模块之间引用，推荐使用相对导入以保证结构灵活、可重定位。
    ```python
    # mypackage/core.py
    from .helpers import format_date      # 导入同包的helpers模块

    # mypackage/subpackage/plugin.py
    from ..core import Database           # 导入上级包中的core模块
    ```

## 标准库核心模块速览

Python 标准库是语言自带的功能宝库，下面按照使用场景列举几个最频繁接触的模块，供你建立初步地图。

| 场景       | 核心模块      | 一句话说明                                                   |
| :--------- | :------------ | :----------------------------------------------------------- |
| 系统交互   | `sys`         | 与解释器交互：`sys.argv`、`sys.exit()`、`sys.path`、`sys.version` |
| 操作系统   | `os`          | 文件路径、目录操作、环境变量 `os.environ`                    |
| 高级路径   | `pathlib`     | 面向对象的文件系统路径操作，比 `os.path` 更现代              |
| 数学       | `math`        | 常用数学函数与常量                                           |
| 随机数     | `random`      | 生成伪随机数：`random.randint()`、`random.choice()`、`random.shuffle()` |
| 日期时间   | `datetime`    | 日期、时间、时间差与格式化                                   |
| 数据序列化 | `json`        | JSON 数据的读写与转换                                        |
| CSV 文件   | `csv`         | 读写 CSV 表格文件                                            |
| 正则表达式 | `re`          | 强大的模式匹配与字符串搜索替换                               |
| 集合相关   | `collections` | 高级容器：`Counter`、`defaultdict`、`OrderedDict`、`deque` 等 |
| 函数工具   | `functools`   | 高阶函数支持：`reduce`、`lru_cache`、`wraps`                 |
| 迭代器工具 | `itertools`   | 高效循环工具：`product`、`permutations`、`chain`、`groupby`  |
| 文件通配符 | `glob`        | 按模式匹配文件路径                                           |
| 日志       | `logging`     | 取代 `print` 的专业日志记录工具                              |
| 命令行解析 | `argparse`    | 构建用户友好的命令行接口                                     |

使用举例（展示几个模块的典型用法）：

```python
# sys: 获取命令行参数
import sys
script_name = sys.argv[0]
arguments = sys.argv[1:]

# pathlib: 路径操作
from pathlib import Path
config = Path.home() / ".myapp" / "config.json"

# datetime: 时间计算
from datetime import datetime, timedelta
today = datetime.now()
yesterday = today - timedelta(days=1)
print(yesterday.strftime("%Y-%m-%d"))

# collections: 高效计数
from collections import Counter
word_counts = Counter("abracadabra")
print(word_counts.most_common(3))
```

!!! tip
    不需要一次性掌握所有模块，但每次想造轮子前，先问自己：标准库是不是已经有现成工具了？

## * 第三方包安装与虚拟环境(venv)

Python 的强大生态很大一部分来自 PyPI（Python Package Index）上几十万的第三方包。pip 是安装它们的官方工具。

### pip 基本使用

在终端中使用 pip 安装包：

```bash
pip install requests          # 安装 requests 包
pip install pandas==2.0.0     # 指定版本
pip install --upgrade pip     # 升级 pip 自身
pip list                      # 列出已安装的包
pip uninstall package_name    # 卸载
```

国内用户可以通过镜像加速下载：

```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple package_name
```

**配置镜像源（永久生效）**

默认 pip 从 PyPI 官方服务器下载，国内用户常因网络原因速度缓慢。可以将镜像源保存为默认配置，一劳永逸。

推荐使用清华大学或阿里云的镜像：

```bash
# 设置为清华镜像（推荐）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

# 或设置为阿里云镜像
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/
```

这条命令会在用户目录生成或修改 `pip.conf`（Windows 为 `pip.ini`），以后所有 `pip install` 都会默认从镜像站拉取。

如果想临时使用某个镜像而不更改全局配置，加 `-i` 参数即可：

```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple package_name
```

验证当前使用的默认镜像：

```bash
pip config list
# 应能看到 global.index-url='https://pypi.tuna.tsinghua.edu.cn/simple'
```

如果需要恢复为官方源：

```bash
pip config unset global.index-url
```

    | 清华大学     | `https://pypi.tuna.tsinghua.edu.cn/simple` |

!!! question "为什么需要虚拟环境？"
    在全局 Python 环境中无节制地安装所有项目的依赖，很快会导致版本冲突（比如，项目 A 需要 Django 3.2，项目 B 需要 Django 4.0）。虚拟环境（virtual environment）给每个项目创建一个 **隔离的 Python 环境**，拥有独立的 `site-packages`，互不干扰。

### **\* 使用 venv 创建虚拟环境**

Python 3.3+ 内置了 `venv` 模块。

**1. 创建环境**

```bash
python -m venv myenv
```

这会在当前目录生成一个 `myenv` 文件夹，里面包含一个隔离的 Python 解释器、一套独立的标准库副本以及一个空的 `site-packages`。

**2. 激活环境**

- Windows CMD：`myenv\Scripts\activate`
- Windows PowerShell：`.\myenv\Scripts\Activate.ps1`
- macOS/Linux：`source myenv/bin/activate`

激活后命令行前会显示环境名称 `(myenv)`，此时运行的 `python` 和 `pip` 都是该环境下的。

**3. 退出环境**

```bash
deactivate
```

**4. 安装依赖到虚拟环境**

激活环境后直接 `pip install` 即可。通常在项目根目录维护一个 `requirements.txt` 文件记录依赖及其版本。

```bash
# 导出当前环境的所有包
pip freeze > requirements.txt

# 根据 requirements.txt 重现环境
pip install -r requirements.txt
```

!!! note "现代工具：pipenv 与 poetry"
    对于更复杂的项目管理，`pipenv` 和 `poetry` 提供了依赖锁定、开发/生产环境分离等更高级的功能。初学阶段掌握 `venv + pip` 已足够，遇到包管理困局时自然会寻求更智能的工具。

### 最佳实践

- **始终在虚拟环境中工作**，不要将项目依赖安装在全局。
- **将虚拟环境目录（如 `venv/`、`.venv/`）加入 `.gitignore`**，不要提交到版本控制系统。
- 保留 `requirements.txt` 或 `pyproject.toml`，让协作者能重建相同的环境。

## 本章小结

模块与包是 Python 软件工程的骨架。`import` 让你能够利用内置库和三方库，`__name__` 守护让文件同时扮演模块和脚本双重角色，包的组织结构和 `__init__.py` 帮你构架清晰的代码边界。标准库提供了开箱即用的“电池”，而 pip 与虚拟环境则保障了依赖管理和项目隔离。下一章我们将直面程序运行中的异常情况——学会优雅地处理错误，告别一崩溃就脸红的时代。
